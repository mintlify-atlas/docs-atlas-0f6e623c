## Overview

Better Auth provides a universal request handler that works with any framework supporting the Web Standard Request/Response API. The handler processes all authentication requests including sign-in, sign-up, OAuth callbacks, and more.

## Handler Function

The `handler` function returned by `betterAuth()` processes incoming requests and returns responses.

```typescript
type Handler = (request: Request) => Promise<Response>
```

### Basic Usage

```typescript
import { auth } from "./auth"

// The handler is a function that accepts Request and returns Response
const response = await auth.handler(request)
```

## Framework Integration

### Next.js App Router

Export the handler for GET and POST methods:

```typescript title="app/api/auth/[...all]/route.ts"
import { auth } from "@/lib/auth"

export const { GET, POST } = auth.handler
```

The catch-all route `[...all]` captures all authentication endpoints:
- `/api/auth/sign-in`
- `/api/auth/sign-up`
- `/api/auth/callback/github`
- etc.

### Next.js Pages Router

Use the handler in API routes:

```typescript title="pages/api/auth/[...all].ts"
import type { NextApiRequest, NextApiResponse } from "next"
import { auth } from "@/lib/auth"

export default async function handler(
	req: NextApiRequest,
	res: NextApiResponse
) {
	// Convert Next.js request to Web Request
	const request = new Request(
		`${process.env.BETTER_AUTH_URL}${req.url}`,
		{
			method: req.method,
			headers: req.headers as HeadersInit,
			body: req.method !== "GET" ? JSON.stringify(req.body) : undefined
		}
	)
	
	const response = await auth.handler(request)
	
	// Convert Web Response to Next.js response
	res.status(response.status)
	response.headers.forEach((value, key) => {
		res.setHeader(key, value)
	})
	res.send(await response.text())
}
```

### SvelteKit

Export the handler in a catch-all route:

```typescript title="src/routes/api/auth/[...all]/+server.ts"
import { auth } from "$lib/auth"

export const { GET, POST } = auth.handler
```

### Astro

Create an API route with a catch-all:

```typescript title="src/pages/api/auth/[...all].ts"
import { auth } from "@/lib/auth"
import type { APIRoute } from "astro"

const handler: APIRoute = async (context) => {
	return auth.handler(context.request)
}

export const GET = handler
export const POST = handler
```

### Remix

Handle auth in a resource route:

```typescript title="app/routes/api.auth.$.tsx"
import type { LoaderFunctionArgs, ActionFunctionArgs } from "@remix-run/node"
import { auth } from "~/lib/auth"

export async function loader({ request }: LoaderFunctionArgs) {
	return auth.handler(request)
}

export async function action({ request }: ActionFunctionArgs) {
	return auth.handler(request)
}
```

### Express

Mount the handler on a path:

```typescript title="server.ts"
import express from "express"
import { auth } from "./auth"

const app = express()

app.all("/api/auth/*", async (req, res) => {
	const request = new Request(`${req.protocol}://${req.get("host")}${req.url}`, {
		method: req.method,
		headers: req.headers as HeadersInit,
		body: req.method !== "GET" ? JSON.stringify(req.body) : undefined
	})
	
	const response = await auth.handler(request)
	
	res.status(response.status)
	response.headers.forEach((value, key) => {
		res.setHeader(key, value)
	})
	res.send(await response.text())
})

app.listen(3000)
```

### Hono

Use the handler directly with Hono:

```typescript title="server.ts"
import { Hono } from "hono"
import { auth } from "./auth"

const app = new Hono()

app.on(["GET", "POST"], "/api/auth/*", async (c) => {
```

### Fastify

```typescript title="server.ts"

	reply.status(response.status)

})

fastify.listen({ port: 3000 })
```

## Custom Handler Configuration

### Base Path

Configure the base path for auth routes:

```typescript
import { betterAuth } from "better-auth"

export const auth = betterAuth({
	basePath: "/auth", // Default: "/api/auth"
	// Routes will be at /auth/sign-in, /auth/sign-up, etc.
})
```

### Disabled Endpoints

Disable specific endpoints:

```typescript
export const auth = betterAuth({
	disabledPaths: [
		"/sign-up",
		"/reset-password"
	]
})
```

### Request Hooks

Add custom logic before/after request processing:

```typescript
export const auth = betterAuth({
	hooks: {
		before: async (ctx) => {
			console.log("Processing:", ctx.request.url)
		},
		after: async (ctx) => {
			console.log("Response:", ctx.response.status)
		}
	}
})
```

## Error Handling

The handler automatically handles errors and returns appropriate responses:

```typescript
// 400 Bad Request - Invalid input
{
	"error": "Invalid email format"
}

// 401 Unauthorized - Not authenticated
{
	"error": "Unauthorized"
}

// 403 Forbidden - No permission
{
	"error": "Forbidden"
}

// 429 Too Many Requests - Rate limited
{
	"error": "Too many requests"
}

// 500 Internal Server Error
{
	"error": "Internal server error"
}
```

### Custom Error Handler

```typescript
export const auth = betterAuth({
	onAPIError: {
		onError: async (error, ctx) => {
			console.error("Auth error:", error)
			// Log to error tracking service
			await logError(error)
		},
		errorURL: "/auth/error"
	}
})
```

## Headers

### Required Headers

The handler expects standard HTTP headers:

```typescript
{
	"content-type": "application/json",
	"cookie": "session_token=...",
	"origin": "https://example.com"
}
```

### CORS Headers

Better Auth automatically handles CORS for trusted origins:

```typescript
export const auth = betterAuth({
	trustedOrigins: [
		"https://example.com",
		"https://*.example.com"
	]
})
```

### Custom Headers

Access custom headers in endpoints:

```typescript
export const myEndpoint = createAuthEndpoint(
	"/custom",
	{ method: "POST" },
	async (ctx) => {
		const customHeader = ctx.request.headers.get("x-custom-header")
		return ctx.json({ customHeader })
	}
)
```

## Testing

### Unit Testing

Mock the handler for testing:

```typescript
import { auth } from "./auth"

test("sign in", async () => {
	const request = new Request("http://localhost:3000/api/auth/sign-in/email", {
		method: "POST",
		headers: { "content-type": "application/json" },
		body: JSON.stringify({
			email: "test@example.com",
			password: "password123"
		})
	})
	
	const response = await auth.handler(request)
	
	expect(response.status).toBe(200)
	const data = await response.json()
	expect(data.user).toBeDefined()
})
```

## Advanced Usage

### Middleware

Add custom middleware before the handler:

```typescript
app.use("/api/auth/*", async (req, res, next) => {
	// Custom logging, rate limiting, etc.
	console.log("Auth request:", req.url)
	next()
})

app.all("/api/auth/*", async (req, res) => {
	const response = await auth.handler(req)
	// Handle response
})
```

### Multiple Auth Instances

Use different auth configurations for different routes:

```typescript
import { betterAuth } from "better-auth"

const adminAuth = betterAuth({
	basePath: "/admin/auth",
	// Admin-specific config
})

const userAuth = betterAuth({
	basePath: "/api/auth",
	// User-specific config
})

app.all("/admin/auth/*", adminAuth.handler)
app.all("/api/auth/*", userAuth.handler)
```

## Notes

<Note>
The handler should only be exposed on the server. Never import or use it on the client side.
</Note>

<Warning>
Make sure your framework supports the Web Standard Request/Response API. Most modern frameworks do, but older versions may require adapters.
</Warning>

<Note>
For serverless environments (Vercel, Netlify), the handler works out of the box. No special configuration needed.
</Note>
