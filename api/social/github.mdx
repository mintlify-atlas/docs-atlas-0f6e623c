# GitHub OAuth

GitHub OAuth allows users to sign in using their GitHub account. It's particularly popular for developer tools and platforms.

## Setup

<Steps>
### Create a GitHub OAuth App

1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Click **OAuth Apps** then **New OAuth App**
3. Fill in the application details:
   - **Application name**: Your app name
   - **Homepage URL**: Your application URL
   - **Authorization callback URL**: `https://example.com/api/auth/callback/github`
4. Click **Register application**

### Get Credentials

1. After creating the app, you'll see your **Client ID**
2. Click **Generate a new client secret** to create a client secret
3. Copy both values immediately (the secret is only shown once)

### Configure Better Auth

Add the credentials to your environment variables:

```bash
GITHUB_CLIENT_ID=your_client_id
GITHUB_CLIENT_SECRET=your_client_secret
```
</Steps>

## Configuration

### Basic Configuration

```ts
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
});
```

### Default Scopes

GitHub provider requests these scopes by default:

- `read:user` - Read user profile data
- `user:email` - Access user email addresses

### Custom Scopes

Request additional permissions:

```ts
github: {
  clientId: process.env.GITHUB_CLIENT_ID!,
  clientSecret: process.env.GITHUB_CLIENT_SECRET!,
  scope: [
    "read:org",      // Read organization membership
    "repo",          // Access repositories
    "public_repo",   // Access public repositories only
  ],
}
```

Available scopes:
- `repo` - Full control of private repositories
- `public_repo` - Access public repositories
- `read:org` - Read organization membership
- `write:org` - Write organization membership
- `admin:org` - Full admin access to organizations
- `gist` - Create gists
- `delete_repo` - Delete repositories

See [GitHub OAuth scopes](https://docs.github.com/en/apps/oauth-apps/building-oauth-apps/scopes-for-oauth-apps) for the complete list.

## Configuration Options

<ParamField path="clientId" type="string" required>
  Your GitHub OAuth App Client ID.
</ParamField>

<ParamField path="clientSecret" type="string" required>
  Your GitHub OAuth App Client Secret.
</ParamField>

<ParamField path="scope" type="string[]">
  Additional OAuth scopes beyond the defaults (`read:user`, `user:email`).
  
  ```ts
  github: {
    clientId: process.env.GITHUB_CLIENT_ID!,
    clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    scope: ["read:org", "repo"],
  }
  ```
</ParamField>

<ParamField path="redirectURI" type="string">
  Custom redirect URI. Must match the callback URL in GitHub OAuth App settings.
  
  Default: `{baseURL}/api/auth/callback/github`
</ParamField>

<ParamField path="prompt" type="'consent' | 'select_account'">
  Control the OAuth consent screen behavior.
  
  ```ts
  github: {
    clientId: process.env.GITHUB_CLIENT_ID!,
    clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    prompt: "consent", // Always show consent screen
  }
  ```
</ParamField>

## User Profile Data

GitHub provides this profile information:

```ts
interface GithubProfile {
  id: string;                    // GitHub user ID
  login: string;                 // Username
  name: string;                  // Display name
  email: string;                 // Primary email
  avatar_url: string;            // Profile picture URL
  bio: string;                   // User bio
  company: string;               // Company name
  location: string;              // Location
  blog: string;                  // Website URL
  twitter_username: string;      // Twitter handle
  hireable: boolean;             // Job seeking status
  public_repos: string;          // Number of public repos
  followers: string;             // Follower count
  following: string;             // Following count
  created_at: string;            // Account creation date
  updated_at: string;            // Last profile update
  two_factor_authentication: boolean; // 2FA enabled
}
```

### Email Handling

GitHub users can have multiple email addresses. Better Auth:

1. First checks the profile's public email
2. If not available, fetches all verified emails
3. Uses the primary verified email
4. Falls back to the first verified email if no primary is set

<Note>
  **Email Verification:** Better Auth automatically checks the `verified` status from GitHub's email API and sets `emailVerified` accordingly.
</Note>

## Client-Side Implementation

```tsx
import { createAuthClient } from "better-auth/client";

const authClient = createAuthClient();

function GitHubSignInButton() {
  const handleSignIn = async () => {
    await authClient.signIn.social({
      provider: "github",
      callbackURL: "/dashboard",
    });
  };

  return (
    <button onClick={handleSignIn}>
      <GitHubIcon />
      Sign in with GitHub
    </button>
  );
}
```

## Advanced Configuration

### Custom User Mapping

Map GitHub profile data to your user schema:

```ts
github: {
  clientId: process.env.GITHUB_CLIENT_ID!,
  clientSecret: process.env.GITHUB_CLIENT_SECRET!,
  mapProfileToUser: (profile) => ({
    // Use GitHub username if name is not available
    name: profile.name || profile.login,
    // Store additional profile data
    username: profile.login,
    bio: profile.bio,
    company: profile.company,
    location: profile.location,
    website: profile.blog,
    twitterHandle: profile.twitter_username,
    // You can add custom fields to your user schema
    githubId: profile.id,
    hasTwo2FA: profile.two_factor_authentication,
  }),
}
```

### Custom User Info Retrieval

Override the default user info fetching:

```ts
github: {
  clientId: process.env.GITHUB_CLIENT_ID!,
  clientSecret: process.env.GITHUB_CLIENT_SECRET!,
  getUserInfo: async (token) => {
    // Fetch user profile
    const profileRes = await fetch("https://api.github.com/user", {
      headers: {
        Authorization: `Bearer ${token.accessToken}`,
        "User-Agent": "better-auth",
      },
    });
    const profile = await profileRes.json();

    // Fetch additional data (e.g., organizations)
    const orgsRes = await fetch("https://api.github.com/user/orgs", {
      headers: {
        Authorization: `Bearer ${token.accessToken}`,
        "User-Agent": "better-auth",
      },
    });
    const orgs = await orgsRes.json();

    return {
      user: {
        id: profile.id,
        name: profile.name,
        email: profile.email,
        image: profile.avatar_url,
        emailVerified: true,
        // Add organization data
        organizations: orgs.map((org: any) => org.login),
      },
      data: { ...profile, organizations: orgs },
    };
  },
}
```

### Login Hints

Pre-fill the GitHub login screen with a username:

```ts
await authClient.signIn.social({
  provider: "github",
  loginHint: "octocat",
  callbackURL: "/dashboard",
});
```

## Using GitHub API

Access the GitHub API using stored OAuth tokens:

```ts
import { auth } from "./auth";

// Get user's GitHub access token
const session = await auth.api.getSession({ headers: request.headers });
if (session?.user) {
  const accounts = await auth.api.getAccounts({ userId: session.user.id });
  const githubAccount = accounts.find(acc => acc.providerId === "github");
  
  if (githubAccount?.accessToken) {
    // Use the token to call GitHub API
    const repos = await fetch("https://api.github.com/user/repos", {
      headers: {
        Authorization: `Bearer ${githubAccount.accessToken}`,
        "User-Agent": "better-auth",
      },
    });
    const data = await repos.json();
  }
}
```

<Note>
  **User-Agent Required:** GitHub API requires a `User-Agent` header on all requests. Better Auth sets this automatically during authentication.
</Note>

## Token Refresh

GitHub access tokens do not expire by default, but you can enable token expiration in your OAuth App settings.

If enabled, Better Auth will automatically refresh expired tokens:

```ts
github: {
  clientId: process.env.GITHUB_CLIENT_ID!,
  clientSecret: process.env.GITHUB_CLIENT_SECRET!,
  // Custom refresh logic (optional)
  refreshAccessToken: async (refreshToken) => {
    const response = await fetch("https://github.com/login/oauth/access_token", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        client_id: process.env.GITHUB_CLIENT_ID,
        client_secret: process.env.GITHUB_CLIENT_SECRET,
        refresh_token: refreshToken,
        grant_type: "refresh_token",
      }),
    });
    return response.json();
  },
}
```

## GitHub Enterprise

To use GitHub Enterprise Server instead of GitHub.com:

```ts
github: {
  clientId: process.env.GITHUB_CLIENT_ID!,
  clientSecret: process.env.GITHUB_CLIENT_SECRET!,
  authorizationEndpoint: "https://github.example.com/login/oauth/authorize",
  // Override getUserInfo to use your enterprise API
  getUserInfo: async (token) => {
    const response = await fetch("https://github.example.com/api/v3/user", {
      headers: {
        Authorization: `Bearer ${token.accessToken}`,
        "User-Agent": "better-auth",
      },
    });
    const profile = await response.json();
    return {
      user: {
        id: profile.id,
        name: profile.name,
        email: profile.email,
        image: profile.avatar_url,
        emailVerified: true,
      },
      data: profile,
    };
  },
}
```

## Troubleshooting

### "redirect_uri_mismatch" Error

**Cause:** The redirect URI in Better Auth doesn't match what's configured in GitHub OAuth App.

**Solution:** 
1. Check your GitHub OAuth App settings
2. Ensure the callback URL matches exactly: `https://yourdomain.com/api/auth/callback/github`
3. Don't use `localhost` in production

### Missing Email Address

**Cause:** User has no public email and hasn't granted `user:email` scope.

**Solution:** Ensure `user:email` scope is requested (it's included by default).

### "Bad credentials" Error

**Cause:** Invalid or expired access token.

**Solution:**
1. Check that your Client ID and Client Secret are correct
2. Regenerate the client secret if needed
3. Ensure tokens are stored securely and not exposed

## Rate Limiting

GitHub API has rate limits:

- **Authenticated requests**: 5,000 requests per hour per user
- **Unauthenticated requests**: 60 requests per hour per IP

Better Auth's authentication flow uses minimal API calls (typically 2-3 per sign-in).
