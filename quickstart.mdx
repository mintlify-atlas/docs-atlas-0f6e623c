<Note>
This quickstart guide will walk you through building a complete authentication system with email/password sign-in, OAuth providers, and session management in a Next.js application.
</Note>

## Prerequisites

Before you begin, make sure you have:

- Node.js 18+ installed
- A Next.js project (or create one with `npx create-next-app@latest`)
- Basic knowledge of TypeScript and React

## Step 1: Install Better Auth

<Steps>
  <Step title="Install the package">
    Add Better Auth to your project:
    
    <CodeGroup>
    ```bash npm
    npm install better-auth
    ```
    
    ```bash pnpm
    pnpm add better-auth
    ```
    
    ```bash yarn
    yarn add better-auth
    ```
    
    ```bash bun
    bun add better-auth
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Install database driver">
    For this quickstart, we'll use SQLite for simplicity:
    
    ```bash
    npm install better-sqlite3
    ```
    
    <Tip>
    For production applications, consider using PostgreSQL or MySQL. See the [database adapters](/adapters) documentation.
    </Tip>
  </Step>
</Steps>

## Step 2: Configure Environment Variables

Create a `.env.local` file in your project root:

```txt title=".env.local"
BETTER_AUTH_SECRET=your-secret-key-here-at-least-32-characters
BETTER_AUTH_URL=http://localhost:3000

# Optional: Add OAuth credentials
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
```

<Tip>
Generate a secure secret:
```bash
openssl rand -base64 32
```
</Tip>

<Accordion title="How to get GitHub OAuth credentials">
  1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
  2. Click **New OAuth App**
  3. Set **Application name** to your app name
  4. Set **Homepage URL** to `http://localhost:3000`
  5. Set **Authorization callback URL** to `http://localhost:3000/api/auth/callback/github`
  6. Click **Register application**
  7. Copy the **Client ID** and generate a **Client Secret**
</Accordion>

## Step 3: Create Auth Instance

Create the server-side auth configuration:

```typescript title="lib/auth.ts"
import { betterAuth } from "better-auth";
import Database from "better-sqlite3";

export const auth = betterAuth({
  database: new Database("./db.sqlite"),
  emailAndPassword: {
    enabled: true,
  },
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
});
```

<Info>
The `auth` object handles all server-side authentication logic, including session management, password hashing, and OAuth flows.
</Info>

## Step 4: Set Up Database Schema

Generate and apply the database schema:

```bash
npx auth@latest generate
npx auth@latest migrate
```

This creates the necessary tables in your SQLite database:
- `user` - Store user information
- `session` - Manage active sessions
- `account` - Link social provider accounts
- `verification` - Handle email verification tokens

## Step 5: Mount API Handler

Create a catch-all API route to handle all authentication requests:

```typescript title="app/api/auth/[...all]/route.ts"
import { auth } from "@/lib/auth";
import { toNextJsHandler } from "better-auth/next-js";

export const { POST, GET } = toNextJsHandler(auth);
```

<Info>
This route handles all authentication endpoints like `/api/auth/sign-in`, `/api/auth/sign-up`, `/api/auth/callback/github`, etc.
</Info>

## Step 6: Create Auth Client

Set up the client-side authentication client:

```typescript title="lib/auth-client.ts"
import { createAuthClient } from "better-auth/react";

export const authClient = createAuthClient();

export const { signIn, signUp, signOut, useSession } = authClient;
```

## Step 7: Build Sign-In Component

Create a sign-in form with email/password and OAuth options:

```tsx title="components/sign-in.tsx"
"use client";

import { useState } from "react";
import { authClient } from "@/lib/auth-client";

export function SignIn() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [error, setError] = useState("");

  const handleSignIn = async (e: React.FormEvent) => {
    e.preventDefault();
    setError("");

    const { data, error } = await authClient.signIn.email({
      email,
      password,
      callbackURL: "/dashboard",
    });

    if (error) {
      setError(error.message);
    }
  };

  const handleGitHubSignIn = async () => {
    await authClient.signIn.social({
      provider: "github",
      callbackURL: "/dashboard",
    });
  };

  return (
    <div className="w-full max-w-md space-y-4">
      <h2 className="text-2xl font-bold">Sign In</h2>
      
      <form onSubmit={handleSignIn} className="space-y-4">
        <div>
          <label htmlFor="email" className="block text-sm font-medium">
            Email
          </label>
          <input
            id="email"
            type="email"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
            className="mt-1 block w-full rounded-md border p-2"
            required
          />
        </div>
        
        <div>
          <label htmlFor="password" className="block text-sm font-medium">
            Password
          </label>
          <input
            id="password"
            type="password"
            value={password}
            onChange={(e) => setPassword(e.target.value)}
            className="mt-1 block w-full rounded-md border p-2"
            required
          />
        </div>
        
        {error && (
          <div className="text-red-600 text-sm">{error}</div>
        )}
        
        <button
          type="submit"
          className="w-full bg-blue-600 text-white rounded-md py-2 hover:bg-blue-700"
        >
          Sign In
        </button>
      </form>
      
      <div className="relative">
        <div className="absolute inset-0 flex items-center">
          <div className="w-full border-t" />
        </div>
        <div className="relative flex justify-center text-sm">
          <span className="bg-white px-2 text-gray-500">Or continue with</span>
        </div>
      </div>
      
      <button
        onClick={handleGitHubSignIn}
        className="w-full border rounded-md py-2 hover:bg-gray-50 flex items-center justify-center gap-2"
      >
        <svg className="w-5 h-5" fill="currentColor" viewBox="0 0 24 24">
```

## Step 8: Build Sign-Up Component

Create a sign-up form to register new users:

```tsx title="components/sign-up.tsx"
"use client";

import { useState } from "react";
import { authClient } from "@/lib/auth-client";

export function SignUp() {
  const [email, setEmail] = useState("");
  const [password, setPassword] = useState("");
  const [name, setName] = useState("");
  const [error, setError] = useState("");

  const handleSignUp = async (e: React.FormEvent) => {
    e.preventDefault();
    setError("");

    const { data, error } = await authClient.signUp.email({
      email,
      password,
      name,
      callbackURL: "/dashboard",
    });

    if (error) {
      setError(error.message);
    }
  };

  return (
    <div className="w-full max-w-md space-y-4">
      <h2 className="text-2xl font-bold">Create Account</h2>
      
      <form onSubmit={handleSignUp} className="space-y-4">
        <div>
          <label htmlFor="name" className="block text-sm font-medium">
            Name
          </label>
          <input
            id="name"
            type="text"
            value={name}
            onChange={(e) => setName(e.target.value)}
            className="mt-1 block w-full rounded-md border p-2"
            required
          />
        </div>
        
        <div>
          <label htmlFor="email" className="block text-sm font-medium">
            Email
          </label>
          <input
            id="email"
            type="email"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
            className="mt-1 block w-full rounded-md border p-2"
            required
          />
        </div>
        
        <div>
          <label htmlFor="password" className="block text-sm font-medium">
            Password
          </label>
          <input
            id="password"
            type="password"
            value={password}
            onChange={(e) => setPassword(e.target.value)}
            className="mt-1 block w-full rounded-md border p-2"
            required
            minLength={8}
          />
        </div>
        
        {error && (
          <div className="text-red-600 text-sm">{error}</div>
        )}
        
        <button
          type="submit"
          className="w-full bg-blue-600 text-white rounded-md py-2 hover:bg-blue-700"
        >
          Create Account
        </button>
      </form>
    </div>
  );
}
```

## Step 9: Create Protected Dashboard

Build a dashboard page that shows user information and allows sign-out:

```tsx title="app/dashboard/page.tsx"
"use client";

import { useSession } from "@/lib/auth-client";
import { redirect } from "next/navigation";

export default function Dashboard() {
  const { data: session, isPending } = useSession();

  if (isPending) {
    return <div>Loading...</div>;
  }

  if (!session) {
    redirect("/sign-in");
  }

  return (
    <div className="min-h-screen p-8">
      <div className="max-w-4xl mx-auto space-y-6">
        <div className="flex items-center justify-between">
          <h1 className="text-3xl font-bold">Dashboard</h1>
          <button
            onClick={() => authClient.signOut()}
            className="px-4 py-2 bg-red-600 text-white rounded-md hover:bg-red-700"
          >
            Sign Out
          </button>
        </div>
        
        <div className="bg-white shadow rounded-lg p-6">
          <h2 className="text-xl font-semibold mb-4">User Information</h2>
          <dl className="space-y-2">
            <div>
              <dt className="text-sm font-medium text-gray-500">Name</dt>
              <dd className="text-lg">{session.user.name}</dd>
            </div>
            <div>
              <dt className="text-sm font-medium text-gray-500">Email</dt>
              <dd className="text-lg">{session.user.email}</dd>
            </div>
            <div>
              <dt className="text-sm font-medium text-gray-500">User ID</dt>
              <dd className="text-lg font-mono text-sm">{session.user.id}</dd>
            </div>
          </dl>
        </div>
      </div>
    </div>
  );
}
```

## What's Happening Under the Hood?

<Steps>
  <Step title="Sign Up Flow">
    1. User submits email, password, and name
    2. Better Auth hashes the password using bcrypt
    3. User record is created in the `user` table
    4. Session is created and stored in `session` table
    5. Session cookie is set in the browser
    6. User is redirected to dashboard
  </Step>
  
  <Step title="Sign In Flow">
    1. User submits email and password
    2. Better Auth looks up user by email
    3. Password is verified against stored hash
    4. New session is created
    5. Session cookie is set
    6. User is redirected to dashboard
  </Step>
  
  <Step title="OAuth Flow">
    1. User clicks "Sign in with GitHub"
    2. Redirected to GitHub for authorization
    3. GitHub redirects back with authorization code
    4. Better Auth exchanges code for access token
    5. User info is fetched from GitHub
    6. User account is created or linked
    7. Session is created and user is signed in
  </Step>
  
  <Step title="Session Management">
    1. `useSession()` hook checks for session cookie
    2. Makes request to `/api/auth/get-session`
    3. Server validates session token
    4. Returns user data if session is valid
    5. Session is automatically refreshed when needed
  </Step>
</Steps>

  ```typescript title="lib/auth.ts"

    database: db,
  ```

## Troubleshooting

<Accordion title="GitHub OAuth not working">
  Make sure your callback URL in GitHub settings matches exactly:
  `http://localhost:3000/api/auth/callback/github`
  
  Also verify that your environment variables are loaded correctly.
</Accordion>

<Accordion title="Session not persisting">
  Check that cookies are enabled in your browser. Better Auth uses HTTP-only cookies for session management.
  
  If using a custom domain, ensure `BETTER_AUTH_URL` matches your actual domain.
</Accordion>

<Accordion title="Database errors">
  Make sure you've run the migration commands:
  ```bash
  npx auth@latest generate
  npx auth@latest migrate
  ```
  
  Check that `db.sqlite` file exists in your project root.
</Accordion>

<Accordion title="TypeScript errors">
  Ensure you have TypeScript 5.0+ installed and your `tsconfig.json` has:
  ```json
  {
    "compilerOptions": {
      "moduleResolution": "bundler",
      "module": "ESNext"
    }
  }
  ```
</Accordion>
