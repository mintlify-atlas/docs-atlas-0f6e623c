---
title: Magic Link
description: Passwordless authentication using magic links sent via email.
---

Magic link authentication provides a passwordless sign-in experience by sending users a unique link via email. When clicked, the link automatically authenticates the user and creates a session.

## Installation

<Steps>
  <Step title="Add the plugin to your auth config">
    Import the magic link plugin and add it to your Better Auth configuration.

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth";
    import { magicLink } from "better-auth/plugins/magic-link";

    export const auth = betterAuth({
      plugins: [
        magicLink({
          sendMagicLink: async ({ email, url, token }, request) => {
            // Send email with the magic link URL
            await sendEmail({
              to: email,
              subject: "Sign in to your account",
              html: `<a href="${url}">Click here to sign in</a>`,
            });
          },
        }),
      ],
    });
    ```
  </Step>

  <Step title="Add the client plugin">
    Add the magic link client plugin to your auth client.

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client";
    import { magicLinkClient } from "better-auth/plugins/magic-link/client";

    export const authClient = createAuthClient({
      plugins: [magicLinkClient()],
    });
    ```
  </Step>
</Steps>

## Usage

### Send Magic Link

To send a magic link to a user's email address, use the `signIn.magicLink` method:

```ts
import { authClient } from "./auth-client";

const { data, error } = await authClient.signIn.magicLink({
  email: "user@example.com",
  callbackURL: "/dashboard",
});

if (data) {
  console.log("Magic link sent successfully");
}
```

**Parameters:**

- `email` (string, required): Email address to send the magic link
- `name` (string, optional): User display name (only used if the user is registering for the first time)
- `callbackURL` (string, optional): URL to redirect after magic link verification (default: `/`)
- `newUserCallbackURL` (string, optional): URL to redirect after new user signup (only used for first-time users)
- `errorCallbackURL` (string, optional): URL to redirect after error

### Verify Magic Link

When a user clicks the magic link in their email, Better Auth automatically handles the verification and creates a session. The user is then redirected to the `callbackURL` you specified.

The magic link URL format is:
```
https://yourdomain.com/api/auth/magic-link/verify?token=TOKEN&callbackURL=/dashboard
```

### New User Signup

By default, magic link authentication will create a new user if the email doesn't exist in your database. The user's email will be automatically verified.

To disable new user signup and only allow existing users to sign in:

```ts title="auth.ts"
magicLink({
  disableSignUp: true,
  sendMagicLink: async ({ email, url, token }, request) => {
    await sendEmail({
      to: email,
      subject: "Sign in to your account",
      html: `<a href="${url}">Click here to sign in</a>`,
    });
  },
})
```

## Configuration

### Basic Configuration

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { magicLink } from "better-auth/plugins/magic-link";

export const auth = betterAuth({
  plugins: [
    magicLink({
      // Required: Function to send magic link emails
      sendMagicLink: async ({ email, url, token }, request) => {
        await sendEmail({
          to: email,
          subject: "Sign in to your account",
          html: `<a href="${url}">Click here to sign in</a>`,
        });
      },
    }),
  ],
});
```

### Advanced Configuration

```ts title="auth.ts"
magicLink({
  // Send magic link implementation
  sendMagicLink: async ({ email, url, token }, request) => {
    await sendEmail({
      to: email,
      subject: "Sign in to your account",
      html: `<a href="${url}">Click here to sign in</a>`,
    });
  },
  
  // Expiration time in seconds (default: 300 = 5 minutes)
  expiresIn: 600, // 10 minutes
  
  // Allowed verification attempts (default: 1)
  allowedAttempts: 3,
  
  // Disable new user signup
  disableSignUp: false,
  
  // Custom token generation
  generateToken: async (email) => {
    // Return a custom token string
    return customTokenGenerator();
  },
  
  // Token storage method (default: "plain")
  storeToken: "hashed", // or "plain"
  
  // Custom rate limiting
  rateLimit: {
    window: 60, // Time window in seconds
    max: 5, // Max requests per window
  },
})
```

### Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `sendMagicLink` | function | **required** | Function to send the magic link email |
| `expiresIn` | number | 300 | Time in seconds until the magic link expires (5 minutes) |
| `allowedAttempts` | number | 1 | Number of allowed verification attempts. Set to `Infinity` for unlimited |
| `disableSignUp` | boolean | false | Disable sign up if user is not found |
| `generateToken` | function | - | Custom function to generate a token |
| `storeToken` | string \| object | "plain" | How to store the token: `"plain"`, `"hashed"`, or custom hasher |
| `rateLimit` | object | `{ window: 60, max: 5 }` | Rate limit configuration |

### Custom Token Hashing

For enhanced security, you can hash the magic link tokens before storing them in the database:

```ts title="auth.ts"
magicLink({
  sendMagicLink: async ({ email, url, token }, request) => {
    await sendEmail({ to: email, html: `<a href="${url}">Sign in</a>` });
  },
  
  // Use hashed token storage
  storeToken: "hashed",
  
  // Or provide a custom hasher
  storeToken: {
    type: "custom-hasher",
    hash: async (token) => {
      return await customHashFunction(token);
    },
  },
})
```

## Error Handling

Magic link authentication can fail for several reasons:

- **Invalid Token**: The token is not found in the database
- **Expired Token**: The token has expired (default: 5 minutes)
- **Attempts Exceeded**: The token has been used too many times
- **New User Signup Disabled**: The email doesn't exist and `disableSignUp` is true

Errors are handled through query parameters in the redirect URL:

```
https://yourdomain.com/callback?error=INVALID_TOKEN
https://yourdomain.com/callback?error=EXPIRED_TOKEN
https://yourdomain.com/callback?error=ATTEMPTS_EXCEEDED
https://yourdomain.com/callback?error=new_user_signup_disabled
```

You can handle these errors in your application:

```tsx
import { useSearchParams } from "next/navigation";

export function MagicLinkCallback() {
  const searchParams = useSearchParams();
  const error = searchParams.get("error");

  if (error) {
    const errorMessages = {
      INVALID_TOKEN: "The magic link is invalid. Please request a new one.",
      EXPIRED_TOKEN: "The magic link has expired. Please request a new one.",
      ATTEMPTS_EXCEEDED: "Too many attempts. Please request a new magic link.",
      new_user_signup_disabled: "This email is not registered. Please sign up first.",
    };

    return <div className="error">{errorMessages[error] || "An error occurred"}</div>;
  }

  return <div>Signing you in...</div>;
}
```

## API Reference

### Endpoints

- `POST /sign-in/magic-link` - Send a magic link to the user's email
- `GET /magic-link/verify` - Verify the magic link token (called automatically)

### Server API

```ts
// Send magic link
auth.api.signInMagicLink({
  body: {
    email: "user@example.com",
    callbackURL: "/dashboard",
  },
});

// Verify magic link
auth.api.magicLinkVerify({
  query: {
    token: "TOKEN",
    callbackURL: "/dashboard",
  },
});
```

### Client API

```ts
// Send magic link
authClient.signIn.magicLink({
  email: "user@example.com",
  callbackURL: "/dashboard",
});

// Verify magic link (automatically called)
authClient.magicLink.verify({
  token: "TOKEN",
  callbackURL: "/dashboard",
});
```

## Security Considerations

<Warning>
  Magic links provide a convenient authentication method but should be implemented carefully to maintain security.
</Warning>

### Best Practices

<Steps>
  <Step title="Use Short Expiration Times">
    Keep magic link expiration times short (5-10 minutes) to minimize the window of vulnerability.
    
    ```ts
    magicLink({
      expiresIn: 300, // 5 minutes
    })
    ```
  </Step>

  <Step title="Limit Verification Attempts">
    Restrict the number of times a magic link can be used to prevent replay attacks.
    
    ```ts
    magicLink({
      allowedAttempts: 1, // Only allow one use
    })
    ```
  </Step>

  <Step title="Hash Tokens in Database">
    Store hashed tokens in the database to protect against database leaks.
    
    ```ts
    magicLink({
      storeToken: "hashed",
    })
    ```
  </Step>

  <Step title="Implement Rate Limiting">
    Prevent abuse by limiting how many magic links can be requested.
    
    ```ts
    magicLink({
      rateLimit: {
        window: 60, // 1 minute
        max: 3, // Max 3 requests per minute
      },
    })
    ```
  </Step>

  <Step title="Use HTTPS">
    Always send magic links over HTTPS to prevent token interception.
  </Step>

  <Step title="Warn Users About Email Security">
    Educate users about the importance of keeping their email accounts secure, as email access grants authentication access.
  </Step>
</Steps>

## Troubleshooting

### Magic Link Not Received

- Check spam/junk folder
- Verify email sending configuration
- Check rate limiting settings
- Review server logs for errors

### Magic Link Expired

- Increase `expiresIn` value if needed
- Request a new magic link

### Invalid Token Error

- Ensure the token hasn't been used already (check `allowedAttempts`)
- Verify token hasn't expired
- Check for URL encoding issues
