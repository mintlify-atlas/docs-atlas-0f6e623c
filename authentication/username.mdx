The Username plugin extends Better Auth's email and password authentication to support username-based sign-in. Users can sign in with either their username or email address, and optionally have a display username that differs from their login username.

## Installation

<Steps>
  <Step title="Add the plugin to your auth config">
    Import the username plugin and add it to your Better Auth configuration.

    ```ts title="auth.ts"
    import { betterAuth } from "better-auth";
    import { username } from "better-auth/plugins/username";

    export const auth = betterAuth({
      emailAndPassword: {
        enabled: true,
      },
      plugins: [username()],
    });
    ```
  </Step>

  <Step title="Migrate the database">
    Run the migration to add the username fields to the user table.

    <Tabs>
      <Tab title="migrate">
        ```bash
        npx @better-auth/cli migrate
        ```
      </Tab>
      <Tab title="generate">
        ```bash
        npx @better-auth/cli generate
        ```
      </Tab>
    </Tabs>

    The plugin adds these fields to the user table:
    - `username` (string, unique): The login username (normalized)
    - `displayUsername` (string, optional): The display username (can have different casing)
  </Step>

  <Step title="Add the client plugin">
    Add the username client plugin to your auth client.

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client";
    import { usernameClient } from "better-auth/plugins/username/client";

    export const authClient = createAuthClient({
      plugins: [usernameClient()],
    });
    ```
  </Step>
</Steps>

## Usage

### Sign Up with Username

When signing up, you can provide a username along with email and password:

```ts
import { authClient } from "./auth-client";

const { data, error } = await authClient.signUp.email({
  email: "user@example.com",
  password: "securePassword123",
  username: "johndoe",
  name: "John Doe",
});
```

**Parameters:**

- `email` (string, required): The email address
- `password` (string, required): The password
- `username` (string, required): The username (will be normalized to lowercase by default)
- `displayUsername` (string, optional): The display username (preserves casing)
- `name` (string, optional): The display name

<Note>
  If you provide `username` but not `displayUsername`, the `displayUsername` will be automatically set to match `username`. Similarly, if you provide `displayUsername` but not `username`, the `username` will be set from `displayUsername`.
</Note>

### Sign In with Username

Users can sign in using their username instead of email:

```ts
import { authClient } from "./auth-client";

const { data, error } = await authClient.signIn.username({
  username: "johndoe",
  password: "securePassword123",
});

if (data) {
  console.log("Signed in:", data.user);
}
```

**Parameters:**

- `username` (string, required): The username
- `password` (string, required): The password
- `rememberMe` (boolean, optional): If false, the user will be signed out when the browser is closed
- `callbackURL` (string, optional): URL to redirect to after sign in

### Check Username Availability

Before sign-up, you can check if a username is available:

```ts
import { authClient } from "./auth-client";

const { data, error } = await authClient.isUsernameAvailable({
  username: "johndoe",
});

if (data?.available) {
  console.log("Username is available");
} else {
  console.log("Username is already taken");
}
```

### Update Username

Users can update their username through the standard user update flow:

```ts
import { authClient } from "./auth-client";

const { data, error } = await authClient.updateUser({
  username: "newusername",
  displayUsername: "NewUsername", // Optional: different casing
});
```

## Configuration

### Basic Configuration

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { username } from "better-auth/plugins/username";

export const auth = betterAuth({
  emailAndPassword: {
    enabled: true,
  },
  plugins: [
    username({
      // Minimum username length (default: 3)
      minUsernameLength: 3,
      
      // Maximum username length (default: 30)
      maxUsernameLength: 30,
    }),
  ],
});
```

### Advanced Configuration

```ts title="auth.ts"
username({
  // Length constraints
  minUsernameLength: 3,
  maxUsernameLength: 30,
  
  // Custom username validation
  usernameValidator: async (username) => {
    // Only allow alphanumeric characters and underscores
    return /^[a-zA-Z0-9_]+$/.test(username);
  },
  
  // Custom display username validation
  displayUsernameValidator: async (displayUsername) => {
    // Allow more characters for display names
    return /^[a-zA-Z0-9_\s-]+$/.test(displayUsername);
  },
  
  // Username normalization (default: lowercase)
  usernameNormalization: (username) => {
    return username.toLowerCase();
  },
  
  // Disable normalization
  usernameNormalization: false,
  
  // Display username normalization (default: no normalization)
  displayUsernameNormalization: (displayUsername) => {
    return displayUsername.trim();
  },
  
  // Validation order
  validationOrder: {
    username: "pre-normalization", // or "post-normalization"
    displayUsername: "pre-normalization",
  },
})
```

### Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `minUsernameLength` | number | 3 | Minimum length of username |
| `maxUsernameLength` | number | 30 | Maximum length of username |
| `usernameValidator` | function | alphanumeric + underscore | Custom username validation function |
| `displayUsernameValidator` | function | - | Custom display username validation function |
| `usernameNormalization` | function \\| false | lowercase | Function to normalize username, or `false` to disable |
| `displayUsernameNormalization` | function \\| false | false | Function to normalize display username |
| `validationOrder.username` | string | "pre-normalization" | When to validate: `"pre-normalization"` or `"post-normalization"` |
| `validationOrder.displayUsername` | string | "pre-normalization" | When to validate display username |

## Username vs Display Username

The plugin supports two username fields:

- **`username`**: The unique, normalized identifier used for login (typically lowercase)
- **`displayUsername`**: The user-facing username with preserved formatting

This allows users to have usernames like:

- Login username: `johndoe` (stored in database)
- Display username: `JohnDoe` (shown in UI)

```ts
await authClient.signUp.email({
  email: "user@example.com",
  password: "password123",
  username: "johndoe",      // Stored as: "johndoe" (normalized)
  displayUsername: "JohnDoe", // Stored as: "JohnDoe" (preserves casing)
});

// User can sign in with normalized username
await authClient.signIn.username({
  username: "johndoe", // or "JohnDoe" - both work
  password: "password123",
});
```

## Validation

### Default Validation

By default, usernames must:
- Be alphanumeric with underscores only: `/^[a-zA-Z0-9_]+$/`
- Be between 3 and 30 characters
- Be unique across all users

### Custom Validation

You can implement custom validation rules:

```ts title="auth.ts"
username({
  usernameValidator: async (username) => {
    // Only allow letters, numbers, underscores, and hyphens
    if (!/^[a-zA-Z0-9_-]+$/.test(username)) {
      return false;
    }
    
    // Don't allow usernames starting with numbers
    if (/^[0-9]/.test(username)) {
      return false;
    }
    
    // Block reserved usernames
    const reserved = ["admin", "root", "system", "api"];
    if (reserved.includes(username.toLowerCase())) {
      return false;
    }
    
    return true;
  },
  
  displayUsernameValidator: async (displayUsername) => {
    // Allow spaces and special characters for display names
    return /^[a-zA-Z0-9_\s-]+$/.test(displayUsername);
  },
})
```

## Normalization

### Default Normalization

By default, usernames are normalized to lowercase:
- Input: `JohnDoe` → Stored: `johndoe`
- This ensures case-insensitive username matching

### Custom Normalization

```ts title="auth.ts"
username({
  // Custom normalization
  usernameNormalization: (username) => {
    return username.toLowerCase().trim();
  },
  
  // Disable normalization (case-sensitive usernames)
  usernameNormalization: false,
  
  // Normalize display username
  displayUsernameNormalization: (displayUsername) => {
    return displayUsername.trim();
  },
})
```

### Validation Order

You can validate usernames before or after normalization:

```ts title="auth.ts"
username({
  validationOrder: {
    // Validate before normalization
    username: "pre-normalization",
    
    // Or validate after normalization
    username: "post-normalization",
  },
})
```

  <Tab title="React">

    import { useState, useEffect } from "react";
    import { authClient } from "./auth-client";

    export function SignUpWithUsername() {
      const [email, setEmail] = useState("");
      const [password, setPassword] = useState("");
      const [username, setUsername] = useState("");
      const [usernameAvailable, setUsernameAvailable] = useState<boolean | null>(null);
      const [error, setError] = useState("");

      // Check username availability on change
      useEffect(() => {
        const checkUsername = async () => {
          if (username.length < 3) {

            return;

          setUsernameAvailable(data?.available ?? false);

        return () => clearTimeout(debounce);

        e.preventDefault();
        setError("");

        if (!usernameAvailable) {
          setError("Username is not available");
          return;
        }

        const { error: signUpError } = await authClient.signUp.email({
          email,
          password,
          username,
          name: username,
        });

          setError(signUpError.message);

          // User signed up successfully

            />

              minLength={8}

### Sign In with Username or Email

```tsx
import { useState } from "react";
import { authClient } from "./auth-client";

export function SignInForm() {
  const [identifier, setIdentifier] = useState("");
  const [password, setPassword] = useState("");
  const [error, setError] = useState("");

  const isEmail = identifier.includes("@");

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setError("");

    const signInMethod = isEmail 
      ? authClient.signIn.email 
      : authClient.signIn.username;

    const payload = isEmail
      ? { email: identifier, password }
      : { username: identifier, password };

    const { error: signInError } = await signInMethod(payload);

    if (signInError) {
      setError(signInError.message);
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <h2>Sign In</h2>
      {error && <div className="error">{error}</div>}
      
      <input
        type="text"
        value={identifier}
        onChange={(e) => setIdentifier(e.target.value)}
        placeholder="Username or Email"
        required
      />
      
      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        placeholder="Password"
        required
      />
      
      <button type="submit">Sign In</button>
      
      <p className="hint">
        {isEmail ? "Signing in with email" : "Signing in with username"}
      </p>
    </form>
  );
}
```

## Schema

The username plugin adds the following fields to the `user` table:

| Field | Type | Description |
|-------|------|-------------|
| `username` | string (unique) | The normalized login username |
| `displayUsername` | string (optional) | The display username with preserved formatting |

## API Reference

### Endpoints

- `POST /sign-in/username` - Sign in with username and password
- `POST /is-username-available` - Check if a username is available
- `POST /sign-up/email` - Sign up with email, password, and username (extended)
- `POST /update-user` - Update user including username (extended)

### Server API

```ts
// Sign in with username
auth.api.signInUsername({
  body: {
    username: "johndoe",
    password: "password123",
  },
});

// Check username availability
auth.api.isUsernameAvailable({
  body: {
    username: "johndoe",
  },
});
```

### Client API

```ts
// Sign in with username
authClient.signIn.username({ username, password })

// Check username availability
authClient.isUsernameAvailable({ username })

// Sign up with username
authClient.signUp.email({ email, password, username, displayUsername })

// Update username
authClient.updateUser({ username, displayUsername })
```

## Error Handling

The username plugin provides specific error codes:

```ts
const { error } = await authClient.signIn.username({
  username: "johndoe",
  password: "password123",
});

if (error) {
  switch (error.message) {
    case "INVALID_USERNAME_OR_PASSWORD":
      console.error("Invalid credentials");
      break;
    case "USERNAME_TOO_SHORT":
      console.error("Username must be at least 3 characters");
      break;
    case "USERNAME_TOO_LONG":
      console.error("Username must be at most 30 characters");
      break;
    case "INVALID_USERNAME":
      console.error("Username contains invalid characters");
      break;
    case "USERNAME_IS_ALREADY_TAKEN":
      console.error("Username is already taken");
      break;
    case "EMAIL_NOT_VERIFIED":
      console.error("Please verify your email first");
      break;
    default:
      console.error("An error occurred");
  }
}
```

## Security Considerations

<Steps>
  <Step title="Use Case-Insensitive Usernames">
    Normalize usernames to lowercase to prevent confusion and potential security issues.
    
    ```ts
    username({
      usernameNormalization: (username) => username.toLowerCase(),
    })
    ```
  </Step>

  <Step title="Validate Username Format">
    Restrict username characters to prevent injection attacks and ensure consistency.
    
    ```ts
    username({
      usernameValidator: (username) => /^[a-zA-Z0-9_]+$/.test(username),
    })
    ```
  </Step>

  <Step title="Reserve System Usernames">
    Block usernames that could be confused with system accounts or API routes.
    
    ```ts
    username({
      usernameValidator: async (username) => {
        const reserved = ["admin", "root", "api", "auth", "system"];
        return !reserved.includes(username.toLowerCase());
      },
    })
    ```
  </Step>

  <Step title="Enforce Length Limits">
    Set appropriate minimum and maximum lengths to prevent abuse.
    
    ```ts
    username({
      minUsernameLength: 3,
      maxUsernameLength: 20,
    })
    ```
  </Step>
</Steps>

## Troubleshooting

### Username Already Taken

Use the `isUsernameAvailable` method to check before attempting sign-up:

```ts
const { data } = await authClient.isUsernameAvailable({ username: "johndoe" });
if (!data?.available) {
  console.error("Username is already taken");
}
```

### Username Validation Fails

Ensure your username meets the validation criteria:
- Default: alphanumeric characters and underscores only
- Check `minUsernameLength` and `maxUsernameLength`
- Verify custom validation rules if implemented

### Sign In Fails with Valid Credentials

- Ensure you're using the normalized username (lowercase by default)
- Check if email verification is required
- Verify the username exists in the database
