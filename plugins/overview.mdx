## Installation

Most plugins follow a similar installation pattern:

<Steps>
  <Step>
    ### Add the server plugin
    
    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { pluginName } from "better-auth/plugins"
    
    export const auth = betterAuth({
      plugins: [
        pluginName({
          // plugin options
        })
      ]
    })
    ```
  </Step>
  
  <Step>
    ### Migrate the database
    
    Most plugins require database changes:
    
    ```bash
    npx auth migrate
    ```
  </Step>
  
  <Step>
    ### Add the client plugin
    
    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"
    import { pluginNameClient } from "better-auth/client/plugins"
    
    export const authClient = createAuthClient({
      plugins: [
        pluginNameClient({
          // plugin options
        })
      ]
    })
    ```
  </Step>
</Steps>

## Plugin Categories

### Authentication Methods

Plugins that add new ways for users to authenticate:

- **Magic Link** - Passwordless email authentication
- **Email OTP** - One-time password via email
- **Phone Number** - SMS-based authentication
- **Username** - Sign in with username
- **Anonymous** - Guest access without credentials

### Security Enhancements

Plugins that improve security:

- **Two-Factor Auth** - TOTP, OTP, and backup codes
- **JWT** - Token-based authentication for APIs
- **Bearer** - Bearer token authentication

### User Management

Plugins for managing users and organizations:

- **Admin** - Administrative user management tools
- **Organization** - Multi-tenant organizations and teams
- **Multi-Session** - Multiple concurrent sessions

### OAuth & Standards

Plugins implementing OAuth and OpenID Connect:

- **OIDC Provider** - Become an OpenID Connect provider
- **Device Authorization** - OAuth 2.0 device flow
- **OAuth Proxy** - Proxy OAuth for development

## Creating Custom Plugins

You can create your own plugins to extend Better Auth. Check out the [Custom Plugins](/plugins/custom) documentation for more information.

## Plugin Best Practices

1. **Install only what you need** - Each plugin adds overhead, so only install plugins you'll actually use
2. **Run migrations** - Most plugins require database changes, so always run migrations after adding plugins
3. **Configure carefully** - Review each plugin's options to ensure proper configuration
4. **Test thoroughly** - Test your authentication flows after adding new plugins
5. **Keep updated** - Update plugins regularly to get the latest features and security fixes

## Need Help?
