Better Auth provides a comprehensive plugin system that allows you to extend the core authentication functionality with additional features. Plugins can add new endpoints, modify database schemas, add hooks, and integrate seamlessly with your authentication flow.

## What are Plugins?

Plugins are modular extensions that enhance Better Auth with specific functionality. Each plugin follows a standard structure and can:

- Add new API endpoints and authentication methods
- Extend the database schema with additional tables and fields
- Hook into the authentication lifecycle (before/after hooks)
- Provide rate limiting and security features
- Add custom middleware and error handling

## Plugin Architecture

Every Better Auth plugin implements the `BetterAuthPlugin` interface, which provides a consistent structure:

```typescript
interface BetterAuthPlugin {
  // Unique identifier for the plugin
  id: string;
  
  // Initialize plugin and modify context
  init?: (ctx: AuthContext) => Promise<{
    context?: Record<string, unknown>;
    options?: Partial<BetterAuthOptions>;
  }>;
  
  // Add new API endpoints
  endpoints?: Record<string, Endpoint>;
  
  // Lifecycle hooks
  hooks?: {
    before?: Array<{ matcher: Function; handler: AuthMiddleware }>;
    after?: Array<{ matcher: Function; handler: AuthMiddleware }>;
  };
  
  // Database schema extensions
  schema?: BetterAuthPluginDBSchema;
  
  // Rate limiting configuration
  rateLimit?: Array<{
    pathMatcher: (path: string) => boolean;
    window: number;
    max: number;
  }>;
  
  // Plugin-specific error codes
  $ERROR_CODES?: Record<string, RawError>;
  
  // Plugin configuration options
  options?: Record<string, any>;
}
```

## Key Plugin Features

### Endpoints

Plugins can add new authentication endpoints that integrate seamlessly with Better Auth:

```typescript
endpoints: {
  signInMagicLink: createAuthEndpoint(
    "/sign-in/magic-link",
    {
      method: "POST",
      body: signInMagicLinkBodySchema,
    },
    async (ctx) => {
      // Endpoint handler logic
    }
  )
}
```

### Database Schema

Plugins can extend the database schema by adding new tables or fields:

```typescript
schema: {
  twoFactor: {
    fields: {
      secret: { type: "string", required: true },
      backupCodes: { type: "string", required: true },
      userId: {
        type: "string",
        required: true,
        references: { model: "user", field: "id" }
      }
    }
  },
  user: {
    fields: {
      twoFactorEnabled: { type: "boolean", defaultValue: false }
    }
  }
}
```

### Hooks

Plugins can hook into the authentication lifecycle to modify behavior:

```typescript
hooks: {
  before: [{
    matcher: (ctx) => ctx.path === "/sign-in/email",
    handler: async (ctx) => {
      // Run before sign-in
    }
  }],
  after: [{
    matcher: (ctx) => ctx.path === "/sign-in/email",
    handler: async (ctx) => {
      // Run after sign-in
    }
  }]
}
```

### Rate Limiting

Plugins can define rate limiting rules for their endpoints:

```typescript
rateLimit: [{
  pathMatcher: (path) => path.startsWith("/two-factor/"),
  window: 10,  // seconds
  max: 3       // max requests per window
}]
```

## Plugin Initialization

The `init` function allows plugins to modify the authentication context and inject custom functionality:

```typescript
init: (ctx: AuthContext) => {
  return {
    context: {
      // Add custom context properties
      customValidator: async (data) => { /* ... */ }
    },
    options: {
      // Modify auth options
      databaseHooks: {
        user: {
          create: {
            before: async (user) => {
              // Modify user before creation
              return { data: { ...user, role: 'user' } };
            }
          }
        }
      }
    }
  };
}
```

## Error Handling

Plugins can define custom error codes that integrate with Better Auth's error system:

```typescript
$ERROR_CODES: {
  INVALID_TOKEN: {
    message: "Invalid verification token",
    status: 400
  },
  EXPIRED_TOKEN: {
    message: "Verification token has expired",
    status: 400
  }
}
```
