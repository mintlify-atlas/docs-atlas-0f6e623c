The OpenAPI plugin automatically generates an OpenAPI 3.1 schema for your Better Auth instance and provides an interactive API reference powered by Scalar.

## Installation

Add the OpenAPI plugin to your authentication setup:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    openAPI()
  ]
})
```

## Features

- **Automatic Schema Generation**: Creates OpenAPI 3.1 schema from your auth configuration
- **Interactive Documentation**: Beautiful API reference UI powered by Scalar
- **Plugin Support**: Includes all enabled plugin endpoints
- **Database Schema**: Documents all database models with field types and validation
- **Standard Responses**: Pre-configured error responses (400, 401, 403, 404, 429, 500)

## Usage

### View Interactive Documentation

Once installed, visit the reference page at:

```
http://localhost:3000/api/auth/reference
```

The interactive documentation allows you to:
- Browse all available endpoints
- View request/response schemas
- Test API calls directly from the browser
- See authentication requirements

### Generate OpenAPI Schema

Retrieve the raw OpenAPI JSON schema:

```ts
const response = await fetch('/api/auth/open-api/generate-schema')
const schema = await response.json()
```

Or using the API:

```ts
import { auth } from "@/lib/auth"

const schema = await auth.api.generateOpenAPISchema()
```

## Configuration

### Custom Reference Path

Change the documentation URL:

```ts title="auth.ts"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    openAPI({
      path: "/docs" // Access at /api/auth/docs
    })
  ]
})
```

### Disable Default Reference Page

Use only the schema generation endpoint:

```ts title="auth.ts"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    openAPI({
      disableDefaultReference: true
    })
  ]
})
```

### Theme Customization

Choose from multiple Scalar themes:

```ts title="auth.ts"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    openAPI({
      theme: "purple" // default | alternate | moon | purple | solarized
                      // bluePlanet | saturn | kepler | mars | deepSpace | laserwave
    })
  ]
})
```

### Content Security Policy (CSP)

Add a nonce for CSP compliance:

```ts title="auth.ts"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    openAPI({
      nonce: "your-csp-nonce"
    })
  ]
})
```

## Generated Schema

### Database Models

All database tables are documented as schemas:

```json
{
  "components": {
    "schemas": {
      "User": {
        "type": "object",
        "properties": {
          "id": { "type": "string" },
          "email": { "type": "string" },
          "name": { "type": "string" },
          "emailVerified": { "type": "boolean" },
          "createdAt": { "type": "string", "format": "date-time" }
        },
        "required": ["email"]
      }
    }
  }
}
```

### Custom Fields

Additional fields are automatically included:

```ts title="auth.ts"
export const auth = betterAuth({
  user: {
    additionalFields: {
      role: {
        type: "string",
        required: true,
        defaultValue: "user"
      }
    }
  },
  plugins: [openAPI()]
})
```

Generates:

```json
{
  "User": {
    "properties": {
      "role": {
        "type": "string",
        "default": "user"
      }
    },
    "required": ["role"]
  }
}
```

### Authentication

All endpoints include security schemes:

```json
{
  "components": {
    "securitySchemes": {
      "bearerAuth": {
        "type": "http",
        "scheme": "bearer",
        "description": "Bearer token authentication"
      },
      "apiKeyCookie": {
        "type": "apiKey",
        "in": "cookie",
        "name": "apiKeyCookie",
        "description": "API Key authentication via cookie"
      }
    }
  }
}
```

## Customizing Endpoints

You can add OpenAPI metadata to custom endpoints:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { createAuthEndpoint } from "better-auth/api"
import { openAPI } from "better-auth/plugins"
import { z } from "zod"

export const auth = betterAuth({
  plugins: [
    openAPI(),
    {
      id: "custom-plugin",
      endpoints: {
        customEndpoint: createAuthEndpoint(
          "/custom",
          {
            method: "POST",
            body: z.object({
              name: z.string(),
              email: z.string().email()
            }),
            metadata: {
              openapi: {
                description: "Create a custom resource",
                tags: ["Custom"],
                operationId: "createCustomResource",
                responses: {
                  "200": {
                    description: "Success",
                    content: {
                      "application/json": {
                        schema: {
                          type: "object",
                          properties: {
                            id: { type: "string" },
                            name: { type: "string" }
                          }
                        }
                      }
                    }
                  }
                }
              }
            }
          },
          async (ctx) => {
            // Handler implementation
            return ctx.json({ id: "123", name: ctx.body.name })
          }
        )
      }
    }
  ]
})
```

## Use Cases

### API Documentation

Provide interactive documentation for your authentication API to frontend developers or third-party integrators.

### Code Generation

Use the OpenAPI schema to generate client SDKs:

```bash
# Generate TypeScript client
npx openapi-typescript http://localhost:3000/api/auth/open-api/generate-schema -o types.ts

# Generate other language clients
npx @openapitools/openapi-generator-cli generate \
  -i http://localhost:3000/api/auth/open-api/generate-schema \
  -g python -o ./client
```

### API Testing

Import the schema into API testing tools:
- Postman
- Insomnia  
- Paw
- Bruno

### Contract Testing

Validate that your API implementation matches the OpenAPI contract.

## Options Reference

<Tip>
All options are optional. The plugin works with sensible defaults.
</Tip>

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `path` | `string` | `"/reference"` | Path to the API reference page |
| `disableDefaultReference` | `boolean` | `false` | Disable the Scalar UI, keep only schema endpoint |
| `theme` | `ScalarTheme` | `"default"` | Scalar theme for the reference page |
| `nonce` | `string` | `undefined` | CSP nonce for inline scripts |

## Example: Full Configuration

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { openAPI } from "better-auth/plugins"

export const auth = betterAuth({
  database: {
    // Your database config
  },
  user: {
    additionalFields: {
      role: {
        type: "string",
        required: true,
        defaultValue: "user"
      },
      department: {
        type: "string",
        required: false
      }
    }
  },
  plugins: [
    openAPI({
      path: "/docs",
      theme: "purple"
    })
  ]
})
```

## Notes

- The plugin follows OpenAPI 3.1 specification
- Zod schemas are automatically converted to JSON Schema
- Optional fields use OpenAPI 3.1 nullable format (type arrays with "null")
- Server-only endpoints (with `SERVER_ONLY` metadata) are excluded from the schema
- Disabled paths (via `disabledPaths` config) are not included
