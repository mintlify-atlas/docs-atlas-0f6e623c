This plugin will soon be deprecated in favor of the OAuth Provider Plugin.
</Callout>

The OIDC Provider Plugin enables you to build and manage your own OpenID Connect (OIDC) provider, granting full control over user authentication without relying on third-party services.

## Features

- **Client Registration**: Register clients to authenticate with your OIDC provider
- **Dynamic Client Registration**: Allow clients to register dynamically
- **Trusted Clients**: Configure hard-coded trusted clients with optional consent bypass
- **Authorization Code Flow**: Support the Authorization Code Flow
- **Public Clients**: Support SPA, mobile apps, CLI tools
- **JWKS Endpoint**: Publish JWKS endpoint for token verification
- **Refresh Tokens**: Issue refresh tokens and handle access token renewal
- **OAuth Consent**: Implement consent screens with bypass option for trusted apps
- **UserInfo Endpoint**: Provide UserInfo endpoint for clients

<Callout type="warn">
This plugin is in active development and may not be suitable for production use.
</Callout>

## Usage

### Register Client

Register a new OIDC client:

```ts
import { authClient } from "@/lib/auth-client"

const { data } = await authClient.oauth2.register({
  redirect_uris: ["https://client.example.com/callback"],
  client_name: "My App",
  logo_uri: "https://client.example.com/logo.png",
  scope: "profile email",
})

if (data) {
  console.log("Client ID:", data.client_id)
  console.log("Client Secret:", data.client_secret)
}
```

### Trusted Clients

Configure trusted clients for first-party applications:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oidcProvider } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oidcProvider({
      loginPage: "/sign-in",
      trustedClients: [
        {
          clientId: "internal-dashboard",
          clientSecret: "secure-secret",
          name: "Internal Dashboard",
          type: "web",
          redirectUrls: ["https://dashboard.company.com/auth/callback"],
          disabled: false,
          skipConsent: true, // Skip consent for trusted client
          metadata: { internal: true }
        }
      ]
    })
  ]
})
```

### UserInfo Endpoint

The `/oauth2/userinfo` endpoint allows clients to retrieve user information:

```ts
const response = await fetch('https://your-domain.com/api/auth/oauth2/userinfo', {
  headers: {
    'Authorization': 'Bearer ACCESS_TOKEN'
  }
})

const userInfo = await response.json()
```

**Returned claims based on scopes:**
- With `openid`: Returns user ID (`sub`)
- With `profile`: Returns `name`, `picture`, `given_name`, `family_name`
- With `email`: Returns `email`, `email_verified`

### Custom Claims

Add custom claims to UserInfo and ID tokens:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oidcProvider } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oidcProvider({
      loginPage: "/sign-in",
      getAdditionalUserInfoClaim: async (user, scopes, client) => {
        const claims: Record<string, any> = {}
        
        if (scopes.includes("profile")) {
          claims.department = user.department
          claims.job_title = user.jobTitle
        }
        
        if (client.metadata?.includeRoles) {
          claims.roles = user.roles
        }
        
        return claims
      }
    })
  ]
})
```

### Consent Screen

Customize the consent screen:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oidcProvider } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oidcProvider({
      consentPage: "/consent"
    })
  ]
})
```

The plugin redirects to your consent page with `consent_code`, `client_id`, and `scope` query parameters.

Accept or deny consent:

```ts
import { authClient } from "@/lib/auth-client"

// Accept consent
await authClient.oauth2.consent({
  accept: true,
  consent_code: consentCode,
})

// Or use cookie-based flow
await authClient.oauth2.consent({
  accept: true,
})
```

## Configuration

### JWKS Endpoint

Integrate with JWT plugin for asymmetric key signing:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oidcProvider, jwt } from "better-auth/plugins"

export const auth = betterAuth({
  disabledPaths: ["/token"],
  plugins: [
    jwt(),
    oidcProvider({
      useJWTPlugin: true,
      loginPage: "/sign-in",
    })
  ]
})
```

### Dynamic Client Registration

Allow public client registration:

```ts title="auth.ts"
const auth = betterAuth({
  plugins: [
    oidcProvider({
      allowDynamicClientRegistration: true,
    })
  ]
})
```

## Schema

The plugin adds these tables:

### OAuth Application
- `id`, `clientId`, `clientSecret`, `name`, `redirectUrls`, `type`, `disabled`, `userId`, `metadata`

### OAuth Access Token
- `id`, `accessToken`, `refreshToken`, `accessTokenExpiresAt`, `refreshTokenExpiresAt`, `clientId`, `userId`, `scopes`

### OAuth Consent
- `id`, `userId`, `clientId`, `scopes`, `consentGiven`

## Options

```ts title="auth.ts"
oidcProvider({
  // Allow dynamic client registration
  allowDynamicClientRegistration: false,
  
  // Custom login page
  loginPage: "/sign-in",
  
  // Custom consent page
  consentPage: "/consent",
  
  // Trusted clients with optional consent bypass
  trustedClients: [],
  
  // Use JWT plugin for ID token signing
  useJWTPlugin: false,
  
  // Additional user info claims
  getAdditionalUserInfoClaim: async (user, scopes, client) => {
    return {}
  },
})
```
