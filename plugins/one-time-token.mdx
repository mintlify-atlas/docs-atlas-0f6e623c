---
title: One-Time Token
description: Generate and verify single-use authentication tokens for secure operations
---

The one-time token (OTT) plugin enables you to generate secure, single-use tokens that can verify a user's session. These tokens are perfect for passwordless authentication flows, secure links, email verification, or any operation requiring temporary session validation.

## Installation

<Steps>
  <Step>
    ### Add the plugin to your auth config
    
    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { oneTimeToken } from "better-auth/plugins"

    export const auth = betterAuth({
      plugins: [
        oneTimeToken()
      ]
    })
    ```
  </Step>
  
  <Step>
    ### Add the client plugin

    ```ts title="auth-client.ts"
    import { createAuthClient } from "better-auth/client"
    import { oneTimeTokenClient } from "better-auth/client/plugins"

    export const authClient = createAuthClient({
      plugins: [
        oneTimeTokenClient()
      ]
    })
    ```
  </Step>
</Steps>

## Usage

### Generate Token

Generate a one-time token for the current authenticated user:

```ts
import { authClient } from "@/lib/auth-client"

const { data } = await authClient.oneTimeToken.generate()

if (data) {
  console.log(data.token) // "abc123xyz..."
  // Share this token with the user (via email, SMS, etc.)
}
```

<Note>
The user must be authenticated to generate a token. The token will be linked to their current session.
</Note>

### Verify Token

Verify a one-time token and establish a session:

```ts
import { authClient } from "@/lib/auth-client"

const { data, error } = await authClient.oneTimeToken.verify({
  token: "abc123xyz..."
})

if (data) {
  console.log("Session established:", data.session)
  console.log("User:", data.user)
  // Session cookie is automatically set
}
```

<Info>
After successful verification:
- The token is immediately deleted (single-use)
- A session cookie is automatically set (unless disabled)
- The user is authenticated
</Info>

## Configuration Options

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      // Token expiration time in minutes
      expiresIn: 3, // Default: 3 minutes
      
      // Disable client-side token generation
      disableClientRequest: false, // Default: false
      
      // Disable automatic session cookie on verification
      disableSetSessionCookie: false, // Default: false
      
      // Custom token generation
      generateToken: async (session, ctx) => {
        // Return a custom token (must be unpredictable)
        return generateSecureToken()
      },
      
      // How to store token in database
      storeToken: "plain", // "plain" | "hashed" | custom hasher
      
      // Automatically set OTT header on new sessions
      setOttHeaderOnNewSession: false // Default: false
    })
  ]
})
```

## Examples

### Email Verification Flow

Send a one-time token via email for account verification:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"
import { sendEmail } from "./email"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      expiresIn: 10, // 10 minutes
      disableClientRequest: true // Only server can generate
    })
  ]
})
```

```ts title="routes/verify-email.ts"
import { auth } from "./auth"

export async function sendVerificationEmail(userId: string) {
  // Get user session
  const session = await auth.api.getSession({
    headers: new Headers({
      cookie: `session=${userId}`
    })
  })
  
  if (session) {
    // Generate token server-side
    const { token } = await auth.api.generateOneTimeToken({
      headers: new Headers({
        cookie: `session=${session.session.token}`
      })
    })
    
    // Create verification URL
    const verifyUrl = `https://app.com/verify?token=${token}`
    
    // Send email
    await sendEmail({
      to: session.user.email,
      subject: "Verify your email",
      html: `Click to verify: <a href="${verifyUrl}">${verifyUrl}</a>`
    })
  }
}
```

Verification page:

```tsx title="app/verify/page.tsx"
import { authClient } from "@/lib/auth-client"
import { useEffect, useState } from "react"
import { useSearchParams } from "next/navigation"

export default function VerifyPage() {
  const searchParams = useSearchParams()
  const token = searchParams.get("token")
  const [status, setStatus] = useState("verifying")
  
  useEffect(() => {
    if (token) {
      verifyToken(token)
    }
  }, [token])
  
  const verifyToken = async (token: string) => {
    const { data, error } = await authClient.oneTimeToken.verify({ token })
    
    if (error) {
      setStatus("error")
    } else {
      setStatus("success")
      // Redirect to dashboard
      window.location.href = "/dashboard"
    }
  }
  
  return (
    <div>
      {status === "verifying" && <p>Verifying your email...</p>}
      {status === "error" && <p>Invalid or expired token</p>}
      {status === "success" && <p>Email verified! Redirecting...</p>}
    </div>
  )
}
```

### Secure Resource Access

Generate tokens for temporary access to protected resources:

```ts title="api/share-document.ts"
import { auth } from "./auth"

export async function shareDocument(
  documentId: string,
  userId: string
) {
  // Generate OTT for the user
  const { token } = await auth.api.generateOneTimeToken({
    headers: new Headers({
      cookie: `session=${userId}`
    })
  })
  
  // Create shareable link
  const shareUrl = `https://app.com/doc/${documentId}?token=${token}`
  
  return { shareUrl, expiresIn: "3 minutes" }
}
```

```ts title="api/view-document.ts"
import { authClient } from "@/lib/auth-client"

export async function viewDocument(documentId: string, token: string) {
  // Verify token and establish session
  const { data, error } = await authClient.oneTimeToken.verify({ token })
  
  if (error) {
    throw new Error("Invalid or expired access token")
  }
  
  // Now authenticated - fetch document
  const document = await fetchDocument(documentId)
  return document
}
```

### Mobile App Authentication

Generate OTT in web app for mobile app login:

```tsx title="components/mobile-login-qr.tsx"
import { authClient } from "@/lib/auth-client"
import { QRCodeSVG } from "qrcode.react"
import { useEffect, useState } from "react"

export function MobileLoginQR() {
  const [token, setToken] = useState("")

  useEffect(() => {
    generateToken()
  }, [])

    if (data) {
      setToken(data.token)
      setLoading(false)
    }
  }
  
  if (loading) {
    return <div>Generating QR code...</div>
  }
  
  return (
    <div>
      <h3>Scan with your mobile app</h3>
      <QRCodeSVG 
        value={JSON.stringify({
          token,
          expiresAt: Date.now() + 3 * 60 * 1000
        })}
        size={256}
      />
      <p>Token expires in 3 minutes</p>
    </div>
  )
}
```

Mobile app verification:

```ts title="mobile/auth.ts"
import { authClient } from "./auth-client"
```

## Advanced Configuration

### Store Token Options

Control how tokens are stored in the database:

**Plain text** (default):
```ts
storeToken: "plain"
```

**Hashed** (more secure):
```ts
storeToken: "hashed"
```

**Custom hasher**:
```ts
storeToken: {
  type: "custom-hasher",
  hash: async (token: string) => {
    return await bcrypt.hash(token, 10)
  }
}
```

<Info>
When using `"hashed"` or a custom hasher, tokens are hashed before storage. The plain token is sent to the user, but only the hash is stored in the database.
</Info>

### Custom Token Generation

Generate tokens with custom format or length:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"
import { generateRandomString } from "better-auth/crypto"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      generateToken: async ({ user, session }, ctx) => {
        // Generate 6-digit numeric code
        const code = Math.floor(100000 + Math.random() * 900000).toString()
        return code
      }
    })
  ]
})
```

<Note>
Custom tokens must be cryptographically secure and unpredictable. The default generator creates 32-character random strings.
</Note>

### Server-Only Token Generation

Prevent clients from generating tokens:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      disableClientRequest: true
    })
  ]
})
```

With this option, only server-side API calls can generate tokens:

```ts
// ✅ Works - server-side
const { token } = await auth.api.generateOneTimeToken({ headers })

// ❌ Fails - client-side
const { error } = await authClient.oneTimeToken.generate()
// error.message: "Client requests are disabled"
```

### Disable Session Cookie

Verify tokens without automatically setting session cookies:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      disableSetSessionCookie: true
    })
  ]
})
```

Useful when you want to manually handle session creation:

```ts
const { data } = await authClient.oneTimeToken.verify({ token })

if (data) {
  // Manually set session or perform additional checks
  await customSessionHandler(data.session)
}
```

### Automatic OTT Header

Automatically attach OTT to new sessions:

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oneTimeToken } from "better-auth/plugins"

export const auth = betterAuth({
  plugins: [
    oneTimeToken({
      setOttHeaderOnNewSession: true
    })
  ]
})
```

The `set-ott` header will be included on sign-in and sign-up responses:

```ts
const response = await authClient.signIn.email({
  email: "user@example.com",
  password: "password"
})

// Access OTT from response headers
const ott = response.headers?.get("set-ott")
```

## Security Considerations

1. **Short Expiration**: Tokens expire quickly (default: 3 minutes)
2. **Single Use**: Tokens are deleted immediately after verification
3. **Session Validation**: Tokens verify the underlying session is still valid
4. **Secure Generation**: Tokens use cryptographically secure random generation
5. **HTTPS Only**: Always use HTTPS in production
6. **Token Storage**: Consider using `"hashed"` storage for sensitive applications
7. **Rate Limiting**: Implement rate limiting on generation endpoints

<Note>
One-time tokens are only as secure as the channel used to transmit them (email, SMS, etc.). Always use secure, trusted delivery methods.
</Note>

## Error Handling

Common errors and how to handle them:

```ts
import { authClient } from "@/lib/auth-client"

const { data, error } = await authClient.oneTimeToken.verify({ token })

if (error) {
  switch (error.message) {
    case "Invalid token":
      // Token doesn't exist or was already used
      console.error("Token not found or already used")
      break
      
    case "Token expired":
      // Token exceeded expiresIn time
      console.error("Token has expired")
      break
      
    case "Session not found":
      // Underlying session was deleted
      console.error("Associated session no longer exists")
      break
      
    case "Session expired":
      // Underlying session has expired
      console.error("User's session has expired")
      break
      
    default:
      console.error("Unknown error:", error.message)
  }
}
```

## Use Cases

1. **Email Verification**: Verify user email addresses with temporary tokens
2. **Password Reset**: Generate tokens for secure password reset flows
3. **Magic Links**: Alternative to traditional magic link authentication
4. **Secure Sharing**: Share protected resources with temporary access
5. **Two-Factor Alternative**: Use as a simplified 2FA method
6. **Mobile Login**: Generate QR codes for mobile app authentication
7. **Guest Access**: Provide temporary access to specific features
8. **API Key Alternative**: Short-lived tokens for temporary API access

## Best Practices

1. **Short Expiration**: Keep token lifetime minimal (3-10 minutes)
2. **Secure Delivery**: Use trusted channels (email, SMS from known numbers)
3. **Clear Communication**: Tell users when tokens expire
4. **Error Messages**: Provide clear feedback on token errors
5. **Rate Limiting**: Limit token generation to prevent abuse
6. **Logging**: Log token generation and verification attempts
7. **Hash Storage**: Use hashed storage for high-security applications
8. **Session Checks**: Verify underlying session hasn't been revoked

## Comparison with Magic Links

| Feature | One-Time Token | Magic Link |
|---------|---------------|------------|
| **Setup** | Simple plugin | Requires email configuration |
| **Flexibility** | Generate anytime | Only for sign-in |
| **Use Cases** | Multiple | Authentication only |
| **Session Required** | Yes | No |
| **Expiration** | Configurable | Configurable |
| **Delivery** | Any method | Email only |

Use **one-time tokens** when you need flexible, temporary session validation for various operations.

Use **magic links** when you need passwordless authentication flows.
