---
title: OAuth Proxy
description: Proxy OAuth requests for development and preview deployments
---

The OAuth Proxy plugin allows you to proxy OAuth requests, making it useful for development and preview deployments where the redirect URL can't be known in advance.

## Installation

<Steps>
  <Step>
    ### Add the plugin to your auth config
    
    ```ts title="auth.ts"
    import { betterAuth } from "better-auth"
    import { oAuthProxy } from "better-auth/plugins"

    export const auth = betterAuth({
      plugins: [
        oAuthProxy({
          productionURL: "https://my-main-app.com", // Optional
          currentURL: "http://localhost:3000", // Optional
        })
      ]
    })
    ```
  </Step>
  
  <Step>
    ### Add redirect URL to OAuth provider

    Add your production app's redirect URL to each social provider:

    ```ts title="auth.ts"
    export const auth = betterAuth({
      plugins: [
        oAuthProxy(),
      ],
      socialProviders: {
        github: {
          clientId: "your-client-id",
          clientSecret: "your-client-secret",
          redirectURI: "https://my-main-app.com/api/auth/callback/github"
        }
      }
    })
    ```
  </Step>
</Steps>

## How It Works

The plugin enables using a single OAuth client across multiple environments:

1. **Preview server initiates OAuth** with production's redirect URI
2. **OAuth provider callbacks** to production server
3. **Production server** exchanges code for tokens and fetches user info
4. **Production encrypts** profile data and redirects to preview server (**no database write**)
5. **Preview server decrypts** profile, creates user/session in its own database

```ts
import { authClient } from "@/lib/auth-client"

await authClient.signIn.social({
  provider: "github",
  callbackURL: "/dashboard"
})
```

The encrypted profile data is passed via URL query parameters and can only be decrypted by servers sharing the same secret.

<Callout type="info">
This plugin is intended for development and preview environments. If `baseURL` and `productionURL` are the same, the plugin won't proxy requests.
</Callout>

## Options

**productionURL**: If this matches `baseURL`, requests won't be proxied. Defaults to `BETTER_AUTH_URL` environment variable.

```ts
oAuthProxy({
  productionURL: "https://my-main-app.com"
})
```

**currentURL**: The application's current URL is automatically determined. Specify manually if auto-detection fails.

```ts
oAuthProxy({
  currentURL: "http://localhost:3000"
})
```

**maxAge**: Maximum age in seconds for encrypted profile payloads. Defaults to `60` seconds.

```ts
oAuthProxy({
  maxAge: 30 // 30 seconds
})
```

Keep this value short (30-60 seconds) to minimize replay attack window while allowing normal OAuth flows.

## Example Setup

### Development

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oAuthProxy } from "better-auth/plugins"

export const auth = betterAuth({
  baseURL: "http://localhost:3000",
  plugins: [
    oAuthProxy({
      productionURL: "https://my-app.com",
    })
  ],
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
      redirectURI: "https://my-app.com/api/auth/callback/github"
    },
    google: {
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      redirectURI: "https://my-app.com/api/auth/callback/google"
    }
  }
})
```

### Preview Deployments

```ts title="auth.ts"
import { betterAuth } from "better-auth"
import { oAuthProxy } from "better-auth/plugins"

const isPreview = process.env.VERCEL_ENV === "preview"

export const auth = betterAuth({
  baseURL: process.env.BETTER_AUTH_URL,
  plugins: [
    ...(isPreview ? [oAuthProxy({
      productionURL: "https://my-app.com"
    })] : [])
  ],
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
      redirectURI: "https://my-app.com/api/auth/callback/github"
    }
  }
})
```

## Security

1. **Encryption**: Profile data is encrypted during transfer
2. **Shared Secret**: Only servers with the same secret can decrypt
3. **Time-Limited**: Payloads expire after `maxAge` seconds
4. **No Production Database**: Production server doesn't write to database
5. **Replay Protection**: Short expiration window prevents replay attacks

## Use Cases

1. **Local Development**: Test OAuth without multiple provider registrations
2. **Preview Deployments**: Test PRs with real OAuth without updating provider settings
3. **Staging Environments**: Use production OAuth config in staging
4. **Multi-Environment**: Same OAuth config across all environments
