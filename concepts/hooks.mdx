## Overview

Better Auth provides two types of hooks that allow you to inject custom logic into the authentication flow:

1. **Database Hooks**: Intercept database operations (create, update, delete)
2. **Request Hooks**: Run before or after API endpoint handlers

Hooks enable powerful customizations like data validation, logging, notifications, and integration with external services.

## Database Hooks

Database hooks run when data is created, updated, or deleted:

### Hook Structure

```typescript
const auth = betterAuth({
  databaseHooks: {
    user: {
      create: {
        before: async (user, ctx) => {
          // Run before user creation
          // Return modified data or false to cancel
        },
        after: async (user, ctx) => {
          // Run after user creation
          // Access the created user
        }
      },
      update: {
        before: async (data, ctx) => {
          // Run before user update
        },
        after: async (user, ctx) => {
          // Run after user update
        }
      },
      delete: {
        before: async (user, ctx) => {
          // Run before user deletion
        },
        after: async (user, ctx) => {
          // Run after user deletion
        }
      }
    },
    session: { /* ... */ },
    account: { /* ... */ }
  }
});
```

### Before Hooks

Before hooks can:
- Validate data
- Transform data
- Cancel operations

```typescript
databaseHooks: {
  user: {
    create: {
      before: async (user, ctx) => {
        // Validate email domain
        if (!user.email.endsWith('@company.com')) {
          return false; // Cancel operation
        }
        
        // Transform data
        return {
          data: {
            ...user,
            role: 'employee',
            department: getDepartmentFromEmail(user.email)
          }
        };
      }
    },
    update: {
      before: async (data, ctx) => {
        // Prevent email changes
        if (data.email) {
          console.warn('Email change attempted');
          return false;
        }
        
        // Add audit trail
        return {
          data: {
            ...data,
            lastModifiedAt: new Date(),
            lastModifiedBy: ctx?.session?.user?.id
          }
        };
      }
    }
  }
}
```

**Source:** `/packages/better-auth/src/db/with-hooks.ts:18`

### After Hooks

After hooks run when operations complete:

```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user, ctx) => {
        // Send welcome email
        await sendEmail({
          to: user.email,
          subject: 'Welcome!',
          template: 'welcome',
          data: { name: user.name }
        });
        
        // Create default preferences
        await db.preferences.create({
          userId: user.id,
          theme: 'light',
          notifications: true
        });
        
        // Log to analytics
        await analytics.track('user_created', {
          userId: user.id,
          email: user.email
        });
      }
    },
    delete: {
      after: async (user, ctx) => {
        // Clean up related data
        await deleteUserFiles(user.id);
        await deleteUserCache(user.id);
        
        // Notify admin
        await notifyAdmin('User deleted', {
          userId: user.id,
          email: user.email
        });
      }
    }
  },
  session: {
    create: {
      after: async (session, ctx) => {
        // Log session creation
        await logAudit('session_created', {
          userId: session.userId,
          ipAddress: session.ipAddress,
          userAgent: session.userAgent
        });
      }
    },
    delete: {
      after: async (session, ctx) => {
        // Log session deletion
        await logAudit('session_deleted', {
          userId: session.userId,
          sessionId: session.id
        });
      }
    }
  }
}
```

### Hook Context

Hooks receive the auth context:

```typescript
interface HookContext {
  session?: {
    user: User;
    session: Session;
  };
  options: BetterAuthOptions;
  adapter: DBAdapter;
  internalAdapter: InternalAdapter;
  logger: Logger;
  // ... other context properties
}
```

Access context in hooks:

```typescript
databaseHooks: {
  user: {
    create: {
      before: async (user, ctx) => {
        // Log with Better Auth's logger
        ctx?.logger?.info('Creating user', { email: user.email });
        
        // Access current session (if authenticated)
        const currentUser = ctx?.session?.user;
        if (currentUser) {
          console.log('User created by:', currentUser.email);
        }
        
        return { data: user };
      }
    }
  }
}
```

### Transaction Support

Hooks run within database transactions when available:

```typescript
// Before hooks run in transaction
databaseHooks: {
  user: {
    create: {
      before: async (user, ctx) => {
        // This runs in transaction
        // If this fails, user creation is rolled back
        await validateUser(user);
        return { data: user };
      },
      after: async (user, ctx) => {
        // After hooks are queued and run after transaction commits
        // If this fails, it doesn't affect user creation
        await sendWelcomeEmail(user.email);
      }
    }
  }
}
```

<Info>
After hooks are queued using `queueAfterTransactionHook` to ensure they run after the transaction commits successfully.
</Info>

## Request Hooks

Request hooks run before or after endpoint handlers:

### Plugin Hooks

Plugins can add request hooks:

```typescript
export const myPlugin = () => {
  return {
    id: "my-plugin",
    hooks: {
      before: [
        {
          matcher: (ctx) => ctx.path === "/sign-up",
          handler: createAuthMiddleware(async (ctx) => {
            // Run before sign-up endpoint
            console.log('Sign-up attempt:', ctx.body?.email);
            
            // Validate request
            if (isBlacklisted(ctx.body?.email)) {
              throw APIError.from('FORBIDDEN', {
                message: 'Email not allowed'
              });
            }
          })
        }
      ],
      after: [
        {
          matcher: (ctx) => ctx.path === "/sign-up",
          handler: createAuthMiddleware(async (ctx) => {
            // Run after sign-up completes
            const user = ctx.context.newSession?.user;
            if (user) {
              await notifySlack('New user signed up', {
                email: user.email
              });
            }
          })
        }
      ]
    }
  };
};
```

**Source:** `/packages/better-auth/src/plugins/anonymous/index.ts:234`

### Matchers

Matchers determine when hooks run:

```typescript
// Exact path match
matcher: (ctx) => ctx.path === "/sign-in/email"

// Prefix match
matcher: (ctx) => ctx.path?.startsWith("/sign-in")

// Multiple paths
matcher: (ctx) => {
  const paths = ["/sign-in/email", "/sign-up/email"];
  return paths.includes(ctx.path || "");
}

// Pattern matching
matcher: (ctx) => /^\/oauth2\/.*$/.test(ctx.path || "")

// Method-based
matcher: (ctx) => ctx.method === "POST"

// Complex logic
matcher: (ctx) => {
  return ctx.path?.startsWith("/admin") && 
         ctx.context.session?.user?.role === "admin";
}

// All requests
matcher: () => true
```

### Hook Handler Context

Request hooks receive the endpoint context:

```typescript
interface EndpointContext {
  request: Request;
  method: string;
  path: string;
  body: any;
  query: any;
  headers: Headers;
  context: AuthContext;
  // ... more properties
}
```

Example using context:

```typescript
hooks: {
  before: [
    {
      matcher: (ctx) => ctx.path?.startsWith("/admin"),
      handler: createAuthMiddleware(async (ctx) => {
        // Check authentication
        const session = ctx.context.session;
        if (!session) {
          throw APIError.from('UNAUTHORIZED', {
            message: 'Authentication required'
          });
        }
        
        // Check authorization
        const user = session.user;
        if (user.role !== 'admin') {
          throw APIError.from('FORBIDDEN', {
            message: 'Admin access required'
          });
        }
        
        // Log access
        await logAccess({
          userId: user.id,
          path: ctx.path,
          method: ctx.method,
          ip: ctx.request.headers.get('x-forwarded-for')
        });
      })
    }
  ]
}
```

## Common Use Cases

### User Onboarding

Create related records when users sign up:

```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user) => {
        // Create user profile
        await db.profile.create({
          userId: user.id,
          bio: '',
          avatar: getDefaultAvatar()
        });
        
        // Add to default organization
        await db.member.create({
          userId: user.id,
          organizationId: DEFAULT_ORG_ID,
          role: 'member'
        });
        
        // Send onboarding email sequence
        await startEmailSequence('onboarding', user.email);
      }
    }
  }
}
```

### Audit Logging

Log all authentication events:

```typescript
hooks: {
  after: [
    {
      matcher: (ctx) => {
        const authPaths = [
          '/sign-in', '/sign-up', '/sign-out',
          '/reset-password', '/verify-email'
        ];
        return authPaths.some(p => ctx.path?.includes(p));
      },
      handler: createAuthMiddleware(async (ctx) => {
        await logAudit({
          event: ctx.path,
          userId: ctx.context.session?.user?.id,
          ip: getIp(ctx.request),
          userAgent: ctx.request.headers.get('user-agent'),
          timestamp: new Date(),
          success: ctx.context.newSession !== null
        });
      })
    }
  ]
}
```

### Data Validation

Enforce business rules:

```typescript
databaseHooks: {
  user: {
    create: {
      before: async (user) => {
        // Enforce email domain
        if (!user.email.match(/@(company\.com|partner\.com)$/)) {
          return false;
        }
        
        // Check duplicate usernames
        const existing = await db.user.findFirst({
          where: { name: user.name }
        });
        if (existing) {
          throw new Error('Username already taken');
        }
        
        return { data: user };
      }
    },
    update: {
      before: async (data, ctx) => {
        // Prevent privilege escalation
        if (data.role === 'admin') {
          const currentUser = ctx?.session?.user;
          if (currentUser?.role !== 'admin') {
            return false;
          }
        }
        
        return { data };
      }
    }
  }
}
```

### External Integrations

Sync with external services:

```typescript
databaseHooks: {
  user: {
    create: {
      after: async (user) => {
        // Create user in CRM
        await crm.createContact({
          email: user.email,
          name: user.name,
          source: 'auth_signup'
        });
        
        // Add to mailing list
        await mailchimp.addSubscriber({
          email: user.email,
          tags: ['new_user']
        });
        
        // Create Stripe customer
        await stripe.customers.create({
          email: user.email,
          metadata: { userId: user.id }
        });
      }
    },
    update: {
      after: async (user) => {
        // Sync changes to external services
        await Promise.all([
          crm.updateContact(user.email, {
            name: user.name,
            emailVerified: user.emailVerified
          }),
          stripe.customers.update(user.stripeCustomerId, {
            name: user.name
          })
        ]);
      }
    },
    delete: {
      before: async (user) => {
        // Archive user in external services
        await crm.archiveContact(user.email);
        await stripe.customers.del(user.stripeCustomerId);
      }
    }
  }
}
```

### Rate Limiting

Implement custom rate limiting:

```typescript
hooks: {
  before: [
    {
      matcher: (ctx) => ctx.path?.includes('/sign-in'),
      handler: createAuthMiddleware(async (ctx) => {
        const ip = getIp(ctx.request);
        const key = `rate_limit:${ip}`;
        
        const count = await redis.incr(key);
        if (count === 1) {
          await redis.expire(key, 60); // 1 minute window
        }
        
        if (count > 5) {
          throw APIError.from('TOO_MANY_REQUESTS', {
            message: 'Too many sign-in attempts'
          });
        }
      })
    }
  ]
}
```

## Hook Best Practices

<AccordionGroup>
  <Accordion title="Keep Hooks Fast">
    Avoid slow operations in hooks:
    
    ```typescript
    // Good: Use background tasks for slow operations
    after: async (user, ctx) => {
      ctx?.runInBackground?.(sendWelcomeEmail(user.email));
    }
    
    // Bad: Slow operations block the response
    after: async (user) => {
      await sleep(5000); // Don't do this!
    }
    ```
  </Accordion>
  
  <Accordion title="Handle Errors Gracefully">
    Catch errors to prevent disrupting normal flow:
    
    ```typescript
    after: async (user) => {
      try {
        await externalService.notify(user);
      } catch (error) {
        logger.error('Failed to notify external service:', error);
        // Don't throw - let user creation succeed
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Use After Hooks for Side Effects">
    Put side effects in after hooks:
    
    ```typescript
    // Good
    after: async (user) => {
      await sendEmail(user.email);
      await updateAnalytics(user.id);
    }
    
    // Bad: Side effects in before hooks can cause issues
    before: async (user) => {
      await sendEmail(user.email); // Sent even if creation fails!
      return { data: user };
    }
    ```
  </Accordion>
  
  <Accordion title="Return Appropriate Values">
    Before hooks should return modified data or false:
    
    ```typescript
    before: async (user) => {
      // Modify data
      return {
        data: { ...user, role: 'user' }
      };
      
      // Or cancel operation
      if (invalid) {
        return false;
      }
    }
    ```
  </Accordion>
  
  <Accordion title="Document Hook Dependencies">
    Make dependencies clear:
    
    ```typescript
    // Document what services hooks depend on
    /**
     * Requires:
     * - Redis connection for rate limiting
     * - SendGrid API key for emails
     * - Stripe API key for customer creation
     */
    databaseHooks: {
      // ...
    }
    ```
  </Accordion>
</AccordionGroup>

## Hook Execution Order

Hooks execute in this order:

1. **Request before hooks** (from plugins)
2. **Endpoint middleware**
3. **Database before hooks** (if database operation)
4. **Database operation** (create/update/delete)
5. **Database after hooks** (queued for after transaction)
6. **Endpoint handler**
7. **Database transaction commits**
8. **Queued after hooks execute**
9. **Request after hooks** (from plugins)

Example flow:

```typescript
// 1. Request before hook
hooks.before[0] → validates request

// 2. Endpoint middleware
sessionMiddleware → checks authentication

// 3. Database before hook
databaseHooks.user.create.before → transforms data

// 4. Database operation
→ INSERT INTO user ...

// 5. Database after hook (queued)
databaseHooks.user.create.after → queued for later

// 6. Endpoint handler
→ returns response

// 7. Transaction commits
→ COMMIT

// 8. Queued hooks execute
→ after hook sends email

// 9. Request after hook
hooks.after[0] → logs request
```
