## Overview

Better Auth provides flexible database integration through a unified adapter interface. It supports multiple databases and ORMs while maintaining a consistent API.

## Database Adapters

Better Auth includes built-in support for several database adapters:

### Built-in Kysely Adapter (Full Mode)

The default adapter uses Kysely for type-safe queries:

```typescript
import { betterAuth } from "better-auth";
import { PostgresDialect } from "kysely";

const auth = betterAuth({
  database: new PostgresDialect({
    pool: new Pool({
      connectionString: process.env.DATABASE_URL
    })
  })
});
```

Supported dialects:
- PostgreSQL
- MySQL
- SQLite
- SQL Server

### ORM Adapters (Minimal Mode)

For minimal bundle sizes, use ORM-specific adapters:

## Database Schema

#### User Table



### Additional Fields

You can extend core tables with additional fields:

```typescript
const auth = betterAuth({
  user: {
    additionalFields: {
      role: {
        type: "string",
        required: true,
        defaultValue: "user",
        input: true, // allow in user input
        output: true // include in API responses
      },
      phoneNumber: {
        type: "string",
        required: false
      }
    }
  },
  session: {
    additionalFields: {
      deviceId: {
        type: "string"
      }
    }
  }
});
```

Additional fields support various options:


## Schema Generation

Better Auth can generate database schemas for your ORM:

### Using CLI

```bash
npx auth generate
```

This generates schema files based on your configuration:

<Tabs>
  <Tab title="Drizzle">
    ```typescript
    // Generated schema.ts
    import { pgTable, text, timestamp, boolean } from "drizzle-orm/pg-core";
    
    export const user = pgTable("user", {
      id: text("id").primaryKey(),
      email: text("email").notNull().unique(),
      emailVerified: boolean("emailVerified").notNull(),
      name: text("name").notNull(),
      role: text("role").notNull().default("user"),
      createdAt: timestamp("createdAt").notNull(),
      updatedAt: timestamp("updatedAt").notNull()
    });
    ```
  </Tab>
  
  <Tab title="Prisma">

### Options

```bash
# Generate to specific file
npx auth generate --output ./db/auth-schema.ts

# Use specific config file
npx auth generate --config ./auth-config.ts

# Auto-accept prompts
npx auth generate --yes
```

## Migrations

### Automatic Migrations (Kysely)

With the built-in Kysely adapter, run migrations automatically:

```bash
npx auth migrate
```

This command:
1. Compares your configuration with the database schema
2. Shows which tables/columns will be added
3. Prompts for confirmation
4. Applies migrations

**Source:** `/packages/cli/src/commands/migrate.ts`

<Warning>
The migrate command only works with the built-in Kysely adapter. For Prisma/Drizzle, use their native migration tools after running `npx auth generate`.
</Warning>

### Manual Migrations

For other adapters:

<Steps>

### Step 1: Generate Schema

```bash
npx auth generate
```

### Step 2: Apply with Your ORM

```bash
# Prisma
npx prisma db push
# or
npx prisma migrate dev

# Drizzle
npx drizzle-kit push
# or
npx drizzle-kit generate
npx drizzle-kit migrate
```

</Steps>

## Internal Adapter

Better Auth uses an internal adapter that wraps the database adapter with additional functionality:

```typescript
interface InternalAdapter {
  // User operations
  createUser(user: Partial<User>): Promise<User>;
  findUserByEmail(email: string): Promise<User | null>;
  updateUser(userId: string, data: Partial<User>): Promise<User>;
  deleteUser(userId: string): Promise<void>;
  
  // Session operations
  createSession(userId: string, options?): Promise<Session>;
  findSession(token: string): Promise<{ session: Session; user: User } | null>;
  updateSession(token: string, data: Partial<Session>): Promise<Session>;
  deleteSession(token: string): Promise<void>;
  listSessions(userId: string): Promise<Session[]>;
  deleteSessions(userId: string): Promise<void>;
  
  // Account operations
  createAccount(account: Partial<Account>): Promise<Account>;
  findAccounts(userId: string): Promise<Account[]>;
  linkAccount(account: Partial<Account>): Promise<Account>;
  
  // OAuth operations
  createOAuthUser(user: Partial<User>, account: Partial<Account>): Promise<{ user: User; account: Account }>;
  
  // Verification operations
  createVerificationValue(data: Partial<Verification>): Promise<Verification>;
  findVerificationValue(identifier: string): Promise<Verification | null>;
  deleteVerificationValue(id: string): Promise<void>;
}
```

**Source:** `/packages/better-auth/src/db/internal-adapter.ts:35`

### Features

The internal adapter provides:

1. **Database Hooks Integration**: Calls before/after hooks for operations
2. **ID Generation**: Generates IDs based on configuration
3. **Secondary Storage**: Caches sessions in Redis/memory
4. **Transaction Support**: Wraps operations in transactions
5. **Date Handling**: Manages createdAt/updatedAt automatically

## Secondary Storage

For improved performance, Better Auth supports secondary storage for sessions:

```typescript
import { betterAuth } from "better-auth";
import { createClient } from "redis";

const redis = createClient();
await redis.connect();

const auth = betterAuth({
  database: /* your database */,
  secondaryStorage: {
    get: async (key) => await redis.get(key),
    set: async (key, value, ttl) => 
      await redis.set(key, value, { EX: ttl }),
    delete: async (key) => await redis.del(key)
  }
});
```

Benefits:
- Faster session lookups
- Reduced database load
- Automatic TTL handling
- Rate limit storage

## Field Parsing

Better Auth parses fields based on configuration:

### Input Parsing

Validates and transforms user input:

```typescript
// In your configuration
user: {
  additionalFields: {
    age: {
      type: "number",
      validator: {
        input: z.number().min(18).max(120)
      },
      transform: {
        input: (value) => parseInt(value)
      }
    }
  }
}
```

When a user signs up, input is validated and transformed automatically.

### Output Parsing

Filters sensitive fields from API responses:

```typescript
// Account table has sensitive tokens
interface Account {
  accessToken?: string; // not included in output
  refreshToken?: string; // not included in output
  userId: string; // included in output
}
```

## Database Hooks

Hooks allow you to intercept database operations:

```typescript
const auth = betterAuth({
  databaseHooks: {
    user: {
      create: {
        before: async (user, ctx) => {
          console.log("Creating user:", user.email);
          // Modify data
          return {
            data: {
              ...user,
              role: user.email.endsWith("@admin.com") ? "admin" : "user"
            }
          };
        },
        after: async (user, ctx) => {
          console.log("User created:", user.id);
          // Send welcome email
          await sendWelcomeEmail(user.email);
        }
      },
      update: {
        before: async (data, ctx) => {
          // Prevent email changes
          if (data.email) {
            return false; // Cancel the operation
          }
        }
      },
      delete: {
        before: async (user, ctx) => {
          // Archive user data before deletion
          await archiveUserData(user.id);
        }
      }
    },
    session: {
      create: {
        after: async (session, ctx) => {
          // Log session creation
          await logSessionCreation(session);
        }
      }
    }
  }
});
```

**Source:** `/packages/better-auth/src/db/with-hooks.ts`

<Info>
Hooks run within transactions when available. After hooks are queued to run after the transaction commits.
</Info>

## Custom Table Names

Rename tables to match your schema:

```typescript
const auth = betterAuth({
  user: {
    modelName: "users"
  },
  session: {
    modelName: "user_sessions"
  }
});
```

Plugins can also customize their table names:

```typescript
import { organization } from "better-auth/plugins";

const auth = betterAuth({
  plugins: [
    organization({
      schema: {
        organization: {
          modelName: "orgs"
        },
        member: {
          modelName: "org_members"
        }
      }
    })
  ]
});
```

## ID Generation

Configure how IDs are generated:

```typescript
const auth = betterAuth({
  advanced: {
    database: {
      generateId: "uuid" // Use UUIDs
      // or
      generateId: "nanoid" // Use nanoid (default)
      // or
      generateId: "serial" // Database auto-increment
      // or
      generateId: false // No ID generation
      // or
      generateId: ({ model, size }) => {
        // Custom ID generation
        return customIdGenerator();
      }
    }
  }
});
```
