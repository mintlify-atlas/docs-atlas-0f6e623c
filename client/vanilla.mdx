# Vanilla JavaScript Client

The vanilla JavaScript client provides framework-agnostic authentication using [nanostores](https://github.com/nanostores/nanostores) for reactive state management. This is the base client that all framework-specific clients are built upon.

## Installation

```bash
npm install better-auth
# or
pnpm add better-auth
# or
yarn add better-auth
```

## Basic Setup

<Steps>

### Create Client Instance

Create a client instance by importing from `better-auth/client`:

```ts title="lib/auth-client.ts"
import { createAuthClient } from "better-auth/client";

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000", // Your backend URL
});
```

### Configure Options

You can configure the client with various options:

```ts title="lib/auth-client.ts"
import { createAuthClient } from "better-auth/client";

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000",
  basePath: "/api/auth", // Default: "/api/auth"
  fetchOptions: {
    credentials: "include", // Send cookies with requests
    onError: (ctx) => {
      console.error("Auth error:", ctx.error);
    },
    onSuccess: (ctx) => {
      console.log("Auth success:", ctx.data);
    },
  },
});
```

</Steps>

## API Reference

### Authentication Methods

All authentication methods are called directly on the client instance:

#### Sign Up

```ts
const { data, error } = await authClient.signUp.email({
  email: "user@example.com",
  password: "securepassword",
  name: "John Doe",
});
```

#### Sign In

```ts
const { data, error } = await authClient.signIn.email({
  email: "user@example.com",
  password: "securepassword",
});
```

#### Social Sign In

```ts
// Redirects to OAuth provider
await authClient.signIn.social({
  provider: "google",
  callbackURL: "/dashboard",
});
```

#### Sign Out

```ts
await authClient.signOut();
```

### Session Management

The vanilla client uses nanostores atoms for reactive session state:

```ts
import { authClient } from "./lib/auth-client";

// Get the session atom
const sessionAtom = authClient.useSession;

// Subscribe to session changes
const unsubscribe = sessionAtom.subscribe((session) => {
  console.log("Session:", session.data);
  console.log("Loading:", session.isPending);
  console.log("Error:", session.error);
});

// Get current session value
const currentSession = sessionAtom.get();

// Manually refetch session
await currentSession.refetch();

// Cleanup subscription
unsubscribe();
```

### Session Object Structure

```ts
interface SessionState {
  data: {
    session: Session | null;
    user: User | null;
  } | null;
  isPending: boolean;
  isRefetching: boolean;
  error: BetterFetchError | null;
  refetch: (queryParams?: { query?: SessionQueryParams }) => Promise<void>;
}
```

### User Management

#### Update User

```ts
const { data, error } = await authClient.updateUser({
  name: "Jane Doe",
  image: "https://example.com/avatar.jpg",
});
```

#### Delete User

```ts
await authClient.deleteUser();
```

#### Change Email

```ts
const { data, error } = await authClient.changeEmail({
  newEmail: "newemail@example.com",
});
```

#### Change Password

```ts
const { data, error } = await authClient.changePassword({
  currentPassword: "oldpassword",
  newPassword: "newpassword",
});
```

### Password Reset

```ts
// Request password reset
const { data, error } = await authClient.forgetPassword({
  email: "user@example.com",
  redirectTo: "/reset-password",
});

// Reset password with token
const { data, error } = await authClient.resetPassword({
  token: "reset-token",
  password: "newpassword",
});
```

### Email Verification

```ts
const { data, error } = await authClient.verifyEmail({
  token: "verification-token",
});
```

## Advanced Usage

### Custom Fetch Implementation

You can provide a custom fetch implementation:

```ts
import { createAuthClient } from "better-auth/client";

const authClient = createAuthClient({
  baseURL: "http://localhost:3000",
  fetchOptions: {
    customFetchImpl: async (url, init) => {
      // Add custom headers or logic
      const headers = new Headers(init?.headers);
      headers.set("X-Custom-Header", "value");
      
      return fetch(url, {
        ...init,
        headers,
      });
    },
  },
});
```

### Store API

The client provides a `$store` API for advanced state management:

```ts
// Manually notify listeners
authClient.$store.notify("$sessionSignal");

// Listen to specific signals
authClient.$store.listen("$sessionSignal", (value, oldValue) => {
  console.log("Session signal changed:", value, oldValue);
});

// Access all atoms
const atoms = authClient.$store.atoms;
```

### Direct Fetch Access

You can access the underlying fetch client:

```ts
// Make custom API calls
const response = await authClient.$fetch("/custom-endpoint", {
  method: "POST",
  body: { data: "value" },
});
```

## Usage with Plugins

When using plugins, their methods are available on the client:

```ts
import { createAuthClient } from "better-auth/client";
import { twoFactorClient } from "better-auth/client/plugins";

const authClient = createAuthClient({
  baseURL: "http://localhost:3000",
  plugins: [twoFactorClient()],
});

// Two-factor methods are now available
await authClient.twoFactor.enable({
  password: "userpassword",
});

const { data } = await authClient.twoFactor.verifyTotp({
  code: "123456",
});
```

## Type Safety

The client provides full TypeScript support with type inference:

```ts
import type { AuthClient } from "better-auth/client";
import type { BetterAuthClientOptions } from "better-auth/client";

// Infer session type
type Session = typeof authClient.$Infer.Session;

// Access error codes
const errorCode = authClient.$ERROR_CODES.INVALID_EMAIL;
```
