## Quick Start Templates

Get started quickly with pre-configured templates:

### Next.js Template

```bash
npx create-next-app@latest my-app --example https://github.com/better-auth/better-auth/tree/main/demo/nextjs
```

### Clone and Run

```bash
git clone https://github.com/better-auth/better-auth
cd better-auth/demo/nextjs
pnpm install
pnpm dev
```

<Note>
  Make sure to copy `.env.example` to `.env` and configure your environment variables before running the examples.
</Note>

## Running Demo Applications

### Next.js Demo

1. Clone the repository
2. Navigate to `demo/nextjs`
3. Copy `.env.example` to `.env` and configure:
   - Database URL (Turso or MySQL)
   - OAuth provider credentials (optional)
   - Email provider settings (optional)
4. Install dependencies: `pnpm install`
5. Run database migrations: `pnpm db:generate`
6. Start the dev server: `pnpm dev`
7. Open [http://localhost:3000](http://localhost:3000)

### Expo Demo

1. Clone the repository
2. Navigate to `demo/expo`
3. Configure environment variables
4. Install dependencies: `pnpm install`
5. Start Expo: `pnpm start`
6. Scan QR code with Expo Go app

### Electron Demo

<Note>
  The Electron demo requires the Next.js demo to be running as the backend server.
</Note>

1. Set up the Next.js demo first (see above)
2. Navigate to `demo/electron`
3. Install dependencies: `pnpm install`
4. Start the app: `pnpm dev`

## Example Features

The Next.js demo showcases all major Better Auth features:

- **Authentication Methods**
  - Email and password authentication
  - Social OAuth providers (Google, GitHub, Discord, etc.)
  - Passwordless magic links
  - Passkeys (WebAuthn)

- **Security Features**
  - Two-factor authentication (TOTP)
  - Email verification
  - Password reset flows
  - Session management
  - Rate limiting

- **Advanced Features**
  - Multi-tenant organizations
  - Role-based access control (RBAC)
  - Member invitations
  - Profile management
  - Stripe integration for subscriptions

## OIDC Client Example

The OIDC client demo shows how to use Better Auth as an OpenID Connect provider:

```bash
cd demo/oidc-client
pnpm install
pnpm dev
```

This example demonstrates:
- OAuth 2.0 / OIDC provider setup
- Custom client applications
- Token management
- Scope-based permissions
