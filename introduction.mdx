## What is Better Auth?

Better Auth is **the most comprehensive authentication framework for TypeScript**. It provides a complete authentication solution that works with any TypeScript framework, whether you're building with Next.js, SvelteKit, Nuxt, Solid, or vanilla JavaScript.

Unlike traditional authentication libraries that require extensive boilerplate code for anything beyond basic features, Better Auth gives you powerful functionality through a simple, plugin-based architecture. Whether you need OAuth, two-factor authentication, passkeys, multi-tenant support, or even enterprise features like SSO, Better Auth lets you focus on building your application instead of reinventing the wheel.

## Why Better Auth?

Authentication in the TypeScript ecosystem has been a half-solved problem. Existing open-source libraries often require significant additional code for anything beyond basic authentication. Rather than pushing third-party services as the only solution, Better Auth provides a comprehensive, self-hosted alternative that gives you complete control.

### Complete Solution

```typescript
import { betterAuth } from "better-auth";
import { twoFactor, passkey, organization } from "better-auth/plugins";

export const auth = betterAuth({
  database: db,
  emailAndPassword: {
    enabled: true,
  },
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
  plugins: [
    twoFactor(),
    passkey(),
    organization(),
  ],
});
```

With just a few lines of configuration, you get:
- Email and password authentication
- GitHub OAuth integration
- Two-factor authentication
- Passkey support
- Multi-tenant organizations

## How It Works

Better Auth consists of two main parts:

<Steps>
  <Step title="Server Instance">
    Create an auth instance on your server that handles all authentication logic, database operations, and API routes.
    
    ```typescript title="lib/auth.ts"
    import { betterAuth } from "better-auth";
    
    export const auth = betterAuth({
      database: db,
      emailAndPassword: { enabled: true },
    });
    ```
  </Step>
  
  <Step title="Client SDK">
    Use the framework-specific client to interact with authentication from your frontend.
    
    ```typescript title="lib/auth-client.ts"
    import { createAuthClient } from "better-auth/react";
    
    export const authClient = createAuthClient();
    ```
  </Step>
  
  <Step title="Start Building">
    Use the client methods to sign in, sign up, and manage sessions.
    
    ```tsx title="components/login.tsx"
    import { authClient } from "@/lib/auth-client";
    
    const { data, error } = await authClient.signIn.email({
      email: "user@example.com",
      password: "password123",
    });
    ```
  </Step>
</Steps>

## Quick Example

Here's a complete authentication flow:

<CodeGroup>
```typescript title="Server (lib/auth.ts)"
import { betterAuth } from "better-auth";
import Database from "better-sqlite3";

export const auth = betterAuth({
  database: new Database("./db.sqlite"),
  emailAndPassword: {
    enabled: true,
  },
  socialProviders: {
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
});
```

```typescript title="Client (lib/auth-client.ts)"
import { createAuthClient } from "better-auth/react";

export const authClient = createAuthClient();

export const { signIn, signOut, signUp, useSession } = authClient;
```

```tsx title="Component (components/auth.tsx)"
import { authClient } from "@/lib/auth-client";

export function Auth() {
  const { data: session } = authClient.useSession();
  
  if (session) {
    return (
      <div>
        <p>Welcome, {session.user.name}!</p>
        <button onClick={() => authClient.signOut()}>Sign Out</button>
      </div>
    );
  }
  
  return (
    <button onClick={() => authClient.signIn.social({ provider: "github" })}>
      Sign in with GitHub
    </button>
  );
}
```
</CodeGroup>
