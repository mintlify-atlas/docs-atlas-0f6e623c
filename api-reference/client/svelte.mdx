## Overview

The Svelte client provides Svelte-specific stores built on top of the core authentication client. It uses nanostores that are compatible with Svelte's store contract for seamless reactivity.

## Installation

```bash
npm install better-auth
```

## Import

```typescript
import { createAuthClient } from "better-auth/svelte"
```

## Setup

```typescript client.ts
import { createAuthClient } from "better-auth/svelte"

const client = createAuthClient({
	baseURL: "http://localhost:3000"
})

export default client
```

## Stores

### useSession

Svelte store for accessing session data with automatic reactivity.

#### Signature

```typescript
function useSession(): Atom<{
	data: Session | null
	error: BetterFetchError | null
	isPending: boolean
	isRefetching: boolean
	refetch: (queryParams?: { query?: SessionQueryParams }) => Promise<void>
}>
```

#### Return Type

Returns a nanostores `Atom` that implements Svelte's store contract:

<ResponseField name="data" type="Session | null">
	The current session data including user and session information. `null` if not authenticated.
	
	```typescript
	type Session = {
		user: {
			id: string
			email: string
			name: string
			emailVerified: boolean
			image?: string
			createdAt: Date
			updatedAt: Date
		}
		session: {
			id: string
			userId: string
			expiresAt: Date
			token: string
			ipAddress?: string
			userAgent?: string
		}
	}
	```
</ResponseField>

<ResponseField name="error" type="BetterFetchError | null">
	Error object if the session fetch failed.
</ResponseField>

<ResponseField name="isPending" type="boolean">
	`true` during the initial session fetch, `false` once complete.
</ResponseField>

<ResponseField name="isRefetching" type="boolean">
	`true` when refetching an existing session, `false` otherwise.
</ResponseField>

<ResponseField name="refetch" type="(queryParams?: { query?: SessionQueryParams }) => Promise<void>">
	Function to manually refetch the session data.
</ResponseField>

#### Usage

```svelte
<script>
	import client from './client'

	const session = client.useSession()
</script>

{#if $session.isPending}
	<div>Loading...</div>
{:else if $session.error}
	<div>Error: {$session.error.message}</div>
{:else if $session.data}
	<div>
		<h1>Welcome, {$session.data.user.name}</h1>
		<p>Email: {$session.data.user.email}</p>
	</div>
{:else}
	<div>Not logged in</div>
{/if}
```

#### Manual Refetch

```svelte
<script>
	import client from './client'

	const session = client.useSession()

	async function handleRefresh() {
		await $session.refetch({
			query: {
				disableCookieCache: true
			}
		})
	}
</script>

<button on:click={handleRefresh}>
	Refresh Session
</button>
```

### Dynamic Plugin Stores

Plugins automatically generate stores based on their atoms.

```typescript
import { createAuthClient } from "better-auth/svelte"
import { organizationClient } from "better-auth/client/plugins"

const client = createAuthClient({
	plugins: [organizationClient()]
})

// Auto-generated store
const organization = client.useActiveOrganization()
```

```svelte
<script>
	import client from './client'

	const organization = client.useActiveOrganization()
</script>

{#if $organization}
	<div>Active Organization: {$organization.name}</div>
{/if}
```

## Authentication Methods

All authentication methods are auto-generated based on your server configuration and are fully type-safe.

### Sign In

```svelte
<script>
	import client from './client'

	let email = ''
	let password = ''
	let error = null

	async function handleSubmit() {
		const result = await client.signIn.email({
			email,
			password
		})

		if (result.error) {
			error = result.error.message
			return
		}

		console.log('Signed in:', result.data)
		error = null
	}
</script>

<form on:submit|preventDefault={handleSubmit}>
	{#if error}
		<div class="error">{error}</div>
	{/if}
	<input
		type="email"
		bind:value={email}
		placeholder="Email"
	/>
	<input
		type="password"
		bind:value={password}
		placeholder="Password"
	/>
	<button type="submit">Sign In</button>
</form>
```

### Sign Up

```svelte
<script>
	import client from './client'

	async function handleSignUp() {
		const { data, error } = await client.signUp.email({
			email: 'user@example.com',
			password: 'password123',
			name: 'John Doe'
		})

		if (error) {
			console.error('Sign up failed:', error)
			return
		}

		console.log('User created:', data)
	}
</script>

<button on:click={handleSignUp}>Sign Up</button>
```

### Sign Out

```svelte
<script>
	import client from './client'
```


## Protected Routes

### SvelteKit Hooks

```typescript hooks.server.ts
import { redirect } from '@sveltejs/kit'
import type { Handle } from '@sveltejs/kit'

export const handle: Handle = async ({ event, resolve }) => {
	// Check authentication on server side
	const session = event.cookies.get('better-auth.session_token')

	if (event.url.pathname.startsWith('/dashboard') && !session) {
		throw redirect(303, '/login')
	}

	return resolve(event)
}
```

### Component-Level Guard

```svelte
<script>
	import { goto } from '$app/navigation'
	import client from './client'

	const session = client.useSession()

	$: if (!$session.isPending && !$session.data) {
		goto('/login')
	}
</script>

{#if $session.isPending}
	<div>Loading...</div>
{:else if $session.data}
	<slot />
{/if}
```

## Reactive Statements

```svelte
<script>
	import client from './client'

	const session = client.useSession()

	$: userName = $session.data?.user.name ?? 'Guest'
	$: isEmailVerified = $session.data?.user.emailVerified ?? false
	$: isLoading = $session.isPending || $session.isRefetching

<div>

	{/if}

	{/if}
```

## Store Subscriptions

You can manually subscribe to stores when needed:

```svelte
<script>
	import { onMount } from 'svelte'
	import client from './client'

	const session = client.useSession()

	onMount(() => {
		const unsubscribe = session.subscribe(value => {
			console.log('Session changed:', value)
		})

		return unsubscribe
	})
</script>
```

## Type Safety

```typescript
import type client from './client'

// Infer types from client
type Session = typeof client.$Infer.Session
type User = Session['user']
type SessionData = Session['session']

// Use in components
export let user: User
```

## Error Handling

```svelte
<script>
	import client from './client'

	let errorMessage = null

	async function handleSignIn(email, password) {
		const { error } = await client.signIn.email({ email, password })

		if (error) {
			if (error.code === client.$ERROR_CODES.INVALID_EMAIL) {
				errorMessage = 'Invalid email address'
			} else if (error.code === client.$ERROR_CODES.INVALID_PASSWORD) {
				errorMessage = 'Invalid password'
			} else {
				errorMessage = 'An error occurred. Please try again.'
			}
			return
		}

		errorMessage = null
	}
</script>

{#if errorMessage}
	<div class="error">{errorMessage}</div>
{/if}
```

## Store Utilities

### Accessing Raw Store Value

```svelte
<script>
	import { get } from 'svelte/store'
	import client from './client'

	const session = client.useSession()

	function logCurrentSession() {
		const currentValue = get(session)
		console.log('Current session:', currentValue)
	}
</script>
```

### Derived Stores

```svelte
<script>
	import { derived } from 'svelte/store'
	import client from './client'

	const session = client.useSession()

	const isAuthenticated = derived(
		session,
		$session => !!$session.data
	)

	const userName = derived(
		session,
		$session => $session.data?.user.name ?? 'Guest'
	)
</script>

{#if $isAuthenticated}
	<p>Welcome back, {$userName}!</p>
{:else}
	<p>Please log in</p>
{/if}
```

## Notes

<Note>
The Svelte client uses nanostores atoms that implement Svelte's store contract, making them compatible with the `$` auto-subscription syntax.
</Note>

<Warning>
Stores are automatically cleaned up when components are destroyed. Manual subscriptions should be unsubscribed in `onDestroy`.
</Warning>

<Info>
The Svelte client works seamlessly with SvelteKit's server-side rendering and client-side navigation.
</Info>
