The Better Auth CLI helps you set up and manage Better Auth in your projects. It provides commands for initialization, schema generation, database migrations, and more.

## Installation

The CLI is included with the `better-auth` package:

```bash
npm install better-auth
# or
pnpm add better-auth
# or
yarn add better-auth
# or
bun add better-auth
```

## Usage

Run commands using `npx`, `pnpm`, or your preferred package manager:

```bash
npx auth <command> [options]
```

## Global Options

Most commands support these common options:

<ParamField path="--cwd" type="string" default="process.cwd()">
  Working directory for the command. Defaults to the current directory.
  
  ```bash
  npx auth generate --cwd /path/to/project
  ```
</ParamField>

<ParamField path="--config" type="string">
  Path to the Better Auth configuration file. Defaults to auto-detection.
  
  ```bash
  npx auth generate --config ./custom/auth.ts
  ```
</ParamField>

<ParamField path="-y, --yes" type="boolean" default="false">
  Automatically answer yes to all prompts.
  
  ```bash
  npx auth migrate --yes
  ```
</ParamField>

## Quick Start

### 1. Initialize Better Auth

```bash
npx auth init
```

This interactive command will:
- Install the `better-auth` package (if needed)
- Set up environment variables
- Create the auth configuration file
- Configure your database
- Set up authentication methods
- Generate schema files
- Create route handlers

### 2. Generate Schema

After initialization or when you modify plugins:

```bash
npx auth generate
```

This generates schema files for your ORM:
- **Prisma**: `prisma/schema.prisma`
- **Drizzle**: `auth-schema.ts` (next to your auth config)
- **Kysely**: Schema is managed internally

### 3. Apply Migrations

For Kysely, run migrations directly:

```bash
npx auth migrate
```

For ORMs:
```bash
# Prisma
npx prisma migrate dev
# or
npx prisma db push

# Drizzle
npx drizzle-kit migrate
# or
npx drizzle-kit push
```

## Configuration Detection

The CLI automatically searches for configuration files in these locations:

<Steps>
  <Step title="Root directory">
    `auth.ts`, `auth.js`, `auth.config.ts`, `auth.config.js`
  </Step>
  
  <Step title="lib/ directory">
    `lib/auth.ts`, `lib/auth.js`, `lib/auth.config.ts`, `lib/auth.config.js`
  </Step>
  
  <Step title="src/ directory">
    `src/auth.ts`, `src/auth.js`, `src/lib/auth.ts`, `src/lib/auth.js`
  </Step>
  
  <Step title="config/ directory">
    `config/auth.ts`, `config/auth.js`
  </Step>
</Steps>

Or specify a custom path:

```bash
npx auth generate --config ./custom-location/auth.ts
```

## Environment Variables

Better Auth requires these environment variables:

```env
BETTER_AUTH_SECRET="your-secret-key"
BETTER_AUTH_URL="http://localhost:3000"
```

Generate a secret:

```bash
npx auth secret
```

## Framework Integration

The CLI detects your framework and generates appropriate files:

<Tabs>
  <Tab title="Next.js">

    - Pages Router: `pages/api/auth/[...all].ts`
  </Tab>
  
  <Tab title="SvelteKit">
    Generates route handler:
    - `src/routes/api/auth/[...all]/+server.ts`
  </Tab>
  
  <Tab title="Nuxt">
    Generates route handler:
    - `server/api/auth/[...all].ts`
  </Tab>
  
  <Tab title="Astro">
    Generates route handler:
    - `src/pages/api/auth/[...all].ts`
  </Tab>
  
  <Tab title="Express/Hono">

  </Tab>
</Tabs>

## Database Support

The CLI supports multiple database configurations:

### ORMs
- Prisma (PostgreSQL, MySQL, SQLite)
- Drizzle (PostgreSQL, MySQL, SQLite)
- MongoDB

### Kysely Dialects
- PostgreSQL (`pg`, `postgres`, `@neondatabase/serverless`, `@vercel/postgres`)
- MySQL (`mysql2`, `@planetscale/database`)
- SQLite (`better-sqlite3`, Bun SQLite, Node SQLite)
- MSSQL (`tedious`)
- Turso/libSQL (`@libsql/client`)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Configuration file not found">
    The CLI searches common locations for `auth.ts` or `auth.js` files. If your file is in a custom location, use the `--config` flag:
    
    ```bash
    npx auth generate --config ./custom/auth-config.ts
    ```
  </Accordion>
  
  <Accordion title="Migration fails">
    For Prisma/Drizzle, use their respective migration tools. The `auth migrate` command only works with Kysely.
    
    ```bash
    # Prisma
    npx prisma migrate dev
    
    # Drizzle
    npx drizzle-kit push
    ```
  </Accordion>
  
  <Accordion title="Schema out of sync">
    Regenerate your schema after adding/removing plugins:
    
    ```bash
    npx auth generate --yes
    ```
  </Accordion>
  
  <Accordion title="Permission errors">
    Make sure you have write permissions in the working directory:
    
    ```bash
    # Check permissions
    ls -la
    
    # Or run from a different directory
    npx auth init --cwd /path/to/project
    ```
  </Accordion>
</AccordionGroup>
