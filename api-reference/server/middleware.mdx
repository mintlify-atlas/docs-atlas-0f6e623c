Better Auth provides built-in middleware for session management, CSRF protection, and origin validation.

## Session Middleware

Middleware to enforce authentication requirements on endpoints.

### sessionMiddleware

<ParamField path="sessionMiddleware" type="AuthMiddleware">
  Requires a valid session to access the endpoint. Throws `UNAUTHORIZED` if no session exists.

  ```typescript
  import { createAuthEndpoint } from "better-auth/api";
  import { sessionMiddleware } from "better-auth/api";

  export const protectedEndpoint = createAuthEndpoint(
    "/protected",
    {
      method: "POST",
      use: [sessionMiddleware]
    },
    async (ctx) => {
      // ctx.context.session is guaranteed to exist
      const { user, session } = ctx.context.session;
      return ctx.json({ user });
    }
  );
  ```

  **Provides:**
  ```typescript
  ctx.context.session: {
    user: User;
    session: Session;
  }
  ```
</ParamField>

### sensitiveSessionMiddleware

<ParamField path="sensitiveSessionMiddleware" type="AuthMiddleware">
  Requires a valid session and **bypasses cookie cache** to ensure the session hasn't been revoked. Use for sensitive operations like password changes or account deletion.

  ```typescript
  import { sensitiveSessionMiddleware } from "better-auth/api";

  export const deleteAccount = createAuthEndpoint(
    "/delete-account",
    {
      method: "POST",
      use: [sensitiveSessionMiddleware]
    },
    async (ctx) => {
      // Session is verified fresh from database
      await deleteUser(ctx.context.session.user.id);
      return ctx.json({ success: true });
    }
  );
  ```

  **Behavior:**
  - Always queries database (ignores `cookieCache`)
  - Prevents use of revoked sessions still cached in cookies
  - Required for security-critical operations
</ParamField>

### freshSessionMiddleware

<ParamField path="freshSessionMiddleware" type="AuthMiddleware">
  Requires a "fresh" session based on the `session.freshAge` configuration. Forces re-authentication if session is too old.

  ```typescript
  import { freshSessionMiddleware } from "better-auth/api";

  export const changePassword = createAuthEndpoint(
    "/change-password",
    {
      method: "POST",
      use: [freshSessionMiddleware]
    },
    async (ctx) => {
      // Session is fresh (recently authenticated)
      // User may need to re-enter password if session is stale
      return ctx.json({ success: true });
    }
  );
  ```

  **Configuration:**
  ```typescript
  // auth.ts
  betterAuth({
    session: {
      freshAge: 60 * 60 * 24, // 1 day (default)
      // Set to 0 to disable freshness checks
    }
  });
  ```

  **Error:** Throws `SESSION_NOT_FRESH` if session is older than `freshAge`.
</ParamField>

### requestOnlySessionMiddleware

<ParamField path="requestOnlySessionMiddleware" type="AuthMiddleware">
  Requires authentication only for client-side requests. Server-side calls (without headers) bypass authentication.

  ```typescript
  import { requestOnlySessionMiddleware } from "better-auth/api";

  export const getProfile = createAuthEndpoint(
    "/profile",
    {
      method: "GET",
      use: [requestOnlySessionMiddleware]
    },
    async (ctx) => {
      // If called from client: session required
      // If called from server: session optional
      const userId = ctx.context.session?.user.id ?? "anonymous";
      return ctx.json({ userId });
    }
  );
  ```
</ParamField>

## CSRF Protection

Better Auth automatically protects against Cross-Site Request Forgery attacks.

### originCheckMiddleware

<ParamField path="originCheckMiddleware" type="AuthMiddleware">
  Automatically applied to all state-modifying requests (POST, PUT, DELETE). Validates:
  
  - Origin/Referer headers match `trustedOrigins`
  - Callback URLs are trusted
  - CSRF tokens when cookies are present

  ```typescript
  // Automatically applied - no manual setup needed
  // Applied to: POST, PUT, DELETE, PATCH
  // Skipped for: GET, OPTIONS, HEAD
  ```

  **Validates:**
  - `callbackURL`
  - `redirectTo`
  - `errorCallbackURL`
  - `newUserCallbackURL`
  - Origin header

  **Configuration:**
  ```typescript
  betterAuth({
    trustedOrigins: [
      "https://example.com",
      "https://*.example.com", // wildcard support
      "http://localhost:3000"
    ],
    advanced: {
      disableCSRFCheck: false, // âš ď¸Ź Don't disable!
      disableOriginCheck: false // âš ď¸Ź May allow open redirects
    }
  });
  ```

  **Error Codes:**
  - `INVALID_ORIGIN`
  - `INVALID_CALLBACK_URL`
  - `INVALID_REDIRECT_URL`
  - `MISSING_OR_NULL_ORIGIN`
</ParamField>

### formCsrfMiddleware

<ParamField path="formCsrfMiddleware" type="AuthMiddleware">
  Enhanced CSRF protection using Fetch Metadata headers for first-login scenarios. Automatically applied to sign-in endpoints.

  ```typescript
  import { formCsrfMiddleware } from "better-auth/api";

  // Already applied to signInEmail and signInSocial
  // Manual usage:
  export const customLogin = createAuthEndpoint(
    "/custom-login",
    {
      method: "POST",
      use: [formCsrfMiddleware]
    },
    async (ctx) => {
      // Protected against cross-site form submissions
      return ctx.json({ success: true });
    }
  );
  ```

  **Protection:**
  - Blocks cross-site navigation login attempts
  - Validates `Sec-Fetch-Site`, `Sec-Fetch-Mode`, `Sec-Fetch-Dest` headers
  - Prevents login CSRF attacks

  **Error:** `CROSS_SITE_NAVIGATION_LOGIN_BLOCKED`
</ParamField>

## Custom Middleware

Create custom middleware for your endpoints.

### createAuthMiddleware

```typescript
import { createAuthMiddleware } from "better-auth/api";

const requireAdmin = createAuthMiddleware(async (ctx) => {
  const session = await getSessionFromCtx(ctx);
  
  if (!session) {
    throw APIError.from("UNAUTHORIZED", {
      message: "Authentication required"
    });
  }

  const user = session.user as User & { role: string };
  
  if (user.role !== "admin") {
    throw APIError.from("FORBIDDEN", {
      message: "Admin access required"
    });
  }

  return {
    session,
    isAdmin: true
  };
});

// Use in endpoint
export const adminEndpoint = createAuthEndpoint(
  "/admin/users",
  {
    method: "GET",
    use: [requireAdmin]
  },
  async (ctx) => {
    // ctx.context.isAdmin is available
    return ctx.json({ users: [] });
  }
);
```

### Plugin Middleware

Plugins can add middleware to specific paths:

```typescript
import type { BetterAuthPlugin } from "better-auth";

export const myPlugin = (): BetterAuthPlugin => {
  return {
    id: "my-plugin",
    middlewares: [
      {
        path: "/custom-path",
        middleware: createAuthMiddleware(async (ctx) => {
          // Custom logic
          return { customData: true };
        })
      }
    ]
  };
};
```

## Middleware Context

Middleware can access and modify the request context:

```typescript
const loggingMiddleware = createAuthMiddleware(async (ctx) => {
  console.log("Request:", {
    path: ctx.request?.url,
    method: ctx.request?.method,
    headers: Object.fromEntries(ctx.headers?.entries() ?? [])
  });

  // Return data to add to context
  return {
    requestId: crypto.randomUUID(),
    timestamp: Date.now()
  };
});

// Use returned data in endpoint
export const endpoint = createAuthEndpoint(
  "/test",
  {
    use: [loggingMiddleware]
  },
  async (ctx) => {
    // Access middleware data
    console.log(ctx.context.requestId);
    console.log(ctx.context.timestamp);
    return ctx.json({ ok: true });
  }
);
```

## Middleware Execution Order

1. **Built-in Router Middleware** (origin checks)
2. **Plugin Middlewares** (path-specific)
3. **Endpoint Middleware** (from `use` array)
4. **Endpoint Handler**

```typescript
betterAuth({
  plugins: [
    myPlugin() // adds middleware
  ],
  hooks: {
    before: async (ctx) => {
      // Runs before middleware
    },
    after: async (ctx) => {
      // Runs after endpoint handler
    }
  }
});
```

## Helper Functions

### getSessionFromCtx

```typescript
import { getSessionFromCtx } from "better-auth/api/routes/session";

const myMiddleware = createAuthMiddleware(async (ctx) => {
  const session = await getSessionFromCtx(ctx, {
    disableCookieCache: true, // Force fresh DB query
    disableRefresh: true      // Don't update session
  });

  if (!session) {
    throw APIError.from("UNAUTHORIZED");
  }

  return { session };
});
```

## Error Handling

Throw errors in middleware to stop request processing:

```typescript
import { APIError } from "better-auth";

const checkPermission = createAuthMiddleware(async (ctx) => {
  const hasPermission = await checkUserPermission(ctx);
  
  if (!hasPermission) {
    throw APIError.from("FORBIDDEN", {
      message: "Insufficient permissions",
      code: "PERMISSION_DENIED"
    });
  }

  return {};
});
```

Available error statuses:
- `UNAUTHORIZED` (401)
- `FORBIDDEN` (403)
- `NOT_FOUND` (404)
- `BAD_REQUEST` (400)
- `INTERNAL_SERVER_ERROR` (500)

## Best Practices

1. **Use `sensitiveSessionMiddleware` for critical operations**
   ```typescript
   // âś… Good
   use: [sensitiveSessionMiddleware]
   
   // âťŚ Bad (cookie cache may be stale)
   use: [sessionMiddleware]
   ```

2. **Don't disable CSRF protection**
   ```typescript
   // âťŚ Never do this
   advanced: { disableCSRFCheck: true }
   ```

3. **Validate trusted origins carefully**
   ```typescript
   // âś… Specific origins
   trustedOrigins: ["https://example.com"]
   
   // âš ď¸Ź Be careful with wildcards
   trustedOrigins: ["https://*.example.com"]
   ```

4. **Return relevant data from middleware**
   ```typescript
   return {
     session, // Make available to endpoint
     userId: session.user.id,
     permissions: await getPermissions(session.user.id)
   };
   ```
