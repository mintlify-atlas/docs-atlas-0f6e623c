Complete reference for Better Auth server-side API endpoints and route handlers.

## Overview

Better Auth provides two ways to interact with authentication:

1. **HTTP Endpoints** - Standard REST API routes handled by `auth.handler`
2. **Server API** - Direct method calls via `auth.api` (no HTTP overhead)

## Authentication Endpoints

### Sign Up with Email

<ParamField path="/sign-up/email" type="POST">
  Create a new user account with email and password.

  **Request Body:**
  ```typescript
  {
    email: string;
    password: string;
    name: string;
    callbackURL?: string;
    rememberMe?: boolean; // default: true
  }
  ```

  **Response:**
  ```typescript
  {
    user: User;
    session: Session;
    redirect: boolean;
    url?: string;
  }
  ```

  **Server API:**
  ```typescript
  const result = await auth.api.signUpEmail({
    body: {
      email: "user@example.com",
      password: "securePassword123",
      name: "John Doe"
    }
  });
  ```
</ParamField>

### Sign In with Email

<ParamField path="/sign-in/email" type="POST">
  Authenticate user with email and password.

  **Request Body:**
  ```typescript
  {
    email: string;
    password: string;
    callbackURL?: string;
    rememberMe?: boolean; // default: true
  }
  ```

  **Response:**
  ```typescript
  {
    user: User;
    token: string;
    redirect: boolean;
    url?: string;
  }
  ```

  **Server API:**
  ```typescript
  const result = await auth.api.signInEmail({
    body: {
      email: "user@example.com",
      password: "securePassword123"
    }
  });
  ```

  **Error Codes:**
  - `INVALID_EMAIL_OR_PASSWORD` - Invalid credentials
  - `EMAIL_NOT_VERIFIED` - Email verification required
  - `EMAIL_PASSWORD_DISABLED` - Feature not enabled
</ParamField>

### Sign In with Social Provider

<ParamField path="/sign-in/social" type="POST">
  Initiate OAuth flow or sign in with ID token.

  **Request Body:**
  ```typescript
  {
    provider: string; // "github" | "google" | ...
    callbackURL?: string;
    newUserCallbackURL?: string;
    errorCallbackURL?: string;
    disableRedirect?: boolean;
    idToken?: {
      token: string;
      nonce?: string;
      accessToken?: string;
      refreshToken?: string;
      expiresAt?: number;
    };
    scopes?: string[];
    requestSignUp?: boolean;
    loginHint?: string;
  }
  ```

  **Response:**
  ```typescript
  // OAuth flow
  { redirect: boolean; url: string; }
  
  // ID token flow
  { redirect: false; token: string; user: User; }
  ```

  **Server API:**
  ```typescript
  const result = await auth.api.signInSocial({
    body: {
      provider: "github",
      callbackURL: "/dashboard"
    }
  });
  ```
</ParamField>

### Sign Out

<ParamField path="/sign-out" type="POST">

  **Server API:**
  ```typescript
  await auth.api.signOut({
    headers: request.headers
  });
  ```

  **Response:**
  ```typescript

  ```
</ParamField>

## Session Management

### Get Session

<ParamField path="/get-session" type="GET | POST">
  Retrieve current session information. POST method requires `deferSessionRefresh` enabled.

  **Query Parameters:**
  ```typescript
  {
    disableCookieCache?: boolean;
    disableRefresh?: boolean;
  }
  ```

  **Response:**
  ```typescript
  {
    user: User;
    session: Session;
    needsRefresh?: boolean; // when deferSessionRefresh enabled
  } | null
  ```

  **Server API:**
  ```typescript
  const session = await auth.api.getSession({
    headers: request.headers,
    query: { disableRefresh: true }
  });
  ```

  The session object includes:
  ```typescript
  {
    session: {
      id: string;
      userId: string;
      token: string;
      expiresAt: Date;
      createdAt: Date;
      updatedAt: Date;
      ipAddress?: string;
      userAgent?: string;
    },
    user: {
      id: string;
      email: string;
      name: string;
      image?: string;
      emailVerified: boolean;
      createdAt: Date;
      updatedAt: Date;
    }
  }
  ```
</ParamField>

## Using Server API

The `auth.api` object provides direct access to all endpoints without HTTP overhead:

```typescript
import { auth } from "./auth";

// In server components, API routes, etc.
export async function getUserSession(request: Request) {
  const session = await auth.api.getSession({
    headers: request.headers
  });

  if (!session) {
    return null;
  }

  return {
    user: session.user,
    session: session.session
  };
}

// Server action example
export async function updateProfile(formData: FormData) {
  "use server";
  
  const name = formData.get("name") as string;
  
  const result = await auth.api.updateUser({
    body: { name },
    headers: await headers() // Next.js
  });

  return result;
}
```

## Error Handling

All endpoints can throw `APIError` with the following structure:

```typescript
{
  status: string; // "UNAUTHORIZED", "BAD_REQUEST", etc.
  message: string;
  code?: string; // Specific error code
}
```

Common error codes:
- `INVALID_EMAIL_OR_PASSWORD`
- `EMAIL_NOT_VERIFIED`
- `UNAUTHORIZED`
- `SESSION_NOT_FRESH`
- `INVALID_TOKEN`
- `USER_NOT_FOUND`
