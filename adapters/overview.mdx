# Database Adapters

Better Auth provides a flexible adapter system that allows you to connect to various databases. Adapters handle all database operations required for authentication, including user management, session handling, and credential storage.

## Available Adapters

Better Auth supports multiple database systems through official and community adapters:

### Official Adapters

- **[Prisma](/adapters/prisma)** - Type-safe ORM for Node.js and TypeScript
- **[Drizzle](/adapters/drizzle)** - Lightweight and performant TypeScript ORM
- **[Kysely](/adapters/kysely)** - Type-safe SQL query builder supporting multiple dialects
- **[MongoDB](/adapters/mongodb)** - Popular NoSQL document database

### Database Dialects via Kysely

The Kysely adapter provides support for numerous SQL databases:

- PostgreSQL
- MySQL
- SQLite
- MS SQL Server
- And many more through [Kysely dialects](/adapters/kysely#supported-dialects)

## Choosing an Adapter

When selecting a database adapter, consider:

<Note>
**Type Safety**: Prisma and Drizzle offer excellent TypeScript support with automatic type generation.

**Performance**: Kysely and Drizzle are lightweight and performant for direct SQL queries.

**Flexibility**: MongoDB is ideal for schema-less data and horizontal scaling.

**Ecosystem**: Prisma has a large ecosystem with extensive tooling and migrations support.
</Note>

## Common Features

All official adapters support:

- **CRUD Operations** - Create, read, update, and delete operations
- **Schema Generation** - CLI tools to generate database schemas
- **Type Safety** - Full TypeScript support
- **Transactions** - Database transaction support where available
- **Custom Field Names** - Ability to customize table and field names
- **Joins (Experimental)** - Performance-optimized database joins (v1.4.0+)
- **Debug Logging** - Optional detailed logging for troubleshooting

## Basic Usage Pattern

All adapters follow a similar integration pattern:

```ts title="auth.ts"
import { betterAuth } from "better-auth";
import { adapterName } from "better-auth/adapters/adapter-name";
import { databaseClient } from "your-database-library";

const client = new databaseClient();

export const auth = betterAuth({
  database: adapterName(client, {
    // adapter-specific options
  }),
});
```

## Schema Management

Better Auth provides a CLI to help manage database schemas:

<Steps>
### Generate Schema

Generate the required database schema based on your configuration:

```bash
npx auth@latest generate
```

### Migrate Database

Apply schema changes to your database:

```bash
npx auth@latest migrate
```

### Update Schema

When adding plugins or customizing fields, regenerate the schema:

```bash
npx auth@latest generate
```
</Steps>

## Performance Optimization

### Experimental Joins

Starting from version 1.4.0, most adapters support experimental joins that can improve performance by 2-3x for operations that fetch related data:

```ts title="auth.ts"
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  database: adapter(client),
  experimental: { joins: true },
});
```

<Warning>
Make sure your database schema has the necessary relations defined. You may need to regenerate your schema with `npx auth@latest generate`.
</Warning>

### Transactions

Most adapters support database transactions for atomic operations:

```ts title="auth.ts"
export const auth = betterAuth({
  database: adapter(client, {
    transaction: true,
  }),
});
```

<Note>
Transactions ensure that multiple database operations either all succeed or all fail together, maintaining data consistency.
</Note>
